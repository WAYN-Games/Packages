    Interface ILocomotionStateMachine<TStateMachine, TStates, TContext, TProfile, TInput, TStateData>

    Interface that defines the structure of a character locomotion state machine, responsible for managing state transitions, physics integration, collision resolution, and other locomotion behaviors of characters.

    Namespace: Wayn.Locomotion.StateMachine
    Assembly: .dll
    Syntax
    public interface ILocomotionStateMachine<TStateMachine, TStates, TContext, TProfile, TInput, TStateData> where TStateMachine : unmanaged, ILocomotionStateMachine<TStateMachine, TStates, TContext, TProfile, TInput, TStateData> where TStates : unmanaged where TContext : unmanaged, ILocomotionContext<TProfile, TInput, TStateData> where TProfile : unmanaged, ILocomotionProfile where TInput : unmanaged, IComponentData where TStateData : unmanaged, IComponentData
    Type Parameters
    Name Description
    TStateMachine

    The type of the locomotion state machine itself, which implements this interface.
    TStates

    The type representing the collection of states managed by the state machine.
    TContext

    The type of the context object that provides state, input data, and profile information necessary for the locomotion logic.
    TProfile

    The type of the locomotion profile containing configuration settings specific to the locomotion logic.
    TInput

    The type of the input data influencing locomotion, such as user inputs or triggers.
    TStateData

    The type of the state data component holding state specific information.
