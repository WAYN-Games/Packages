    Generics and Source Generator

    This package relies on generic types and source generators to reduce the boilerplate code needed to implement the character locomotion. Understanding the which generic types exists and what is source generated is key for your comprehension of the package inner working and to enable customisation fo the behavior.

    Generics

    TStateMachine: The type of the locomotion state machine itself, which implements this interface.

    TStates: The type representing the collection of states managed by the state machine.

    TContext: The type of the context object that provides state, input data, and profile information necessary for the locomotion logic.

    TProfile: The type of the locomotion profile containing configuration settings specific to the locomotion logic.

    TInput: The type of the input data influencing locomotion, such as user inputs or triggers.

    TStateData: The type of the state data component holding state specific information.

    Source Generated Content

    All source generated content is generated from your list of states. For the generation to work the enum need to be marked with the attribute LocomotionStateMachine. Each enum value needs to be marked with the LocomotionState(TStates) attribute. All generated types are named using the concrete type of TStates as a prefix. As part of this documentation they will be reference using the #TStates#. All generated types are generated as partial types to allow their extension.

    StateMachine

    The state machine is source generated by picking up the LocomotionStateMachine. As part of the attribute you have to define your specific implementation of the generic types TContext, TProfile, TInput, TStateData. This allows the source generator to create a partial struct to hold the current state of the state machine :

        public partial struct #TStates#StateMachine : IComponentData,ILocomotionStateMachine<#TStates#StateMachine, #TStates#LocomotionStates, #TStates#Context, #TStates#Profile, #TStates#Input, #TStates#StateData>

    For each of the method defined by the interface, the generator will implement switch statements from the enum values marked with LocomotionState(TStates) attribute. The generated #TState#StateMachine will be added to the character entity by the baking system and stores the current state (TState) of the state machine.

    Input reader system

    Using the #TState#InputReaderSystem TInput defined in the LocomotionStateMachine attribute, the pâckage will generate two input systems :

    • PlayerInputManagerSystem : in charge of initialising the player input asset and capturing the player input from the asset
        public partial struct #TState#PlayerInputManagerSystem : ISystem
    • InputReaderSystem : in charge of applying the player input to the player controlled characters
        public partial struct #TState#InputReaderSystem : ISystem

    Movement System

    This is the system that schedule the execution of the state machine. For that it uses all the generic types defined by the LocomotionStateMachine attribute to generate a ISystem:

        public partial struct BasicLocomotionStatesLocomotionSystem : ISystem
