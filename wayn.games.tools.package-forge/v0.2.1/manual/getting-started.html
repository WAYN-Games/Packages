    Getting Started

    This guide walks you through installing DocTools, initializing documentation for a package, and running your first build.

    Prerequisites

    Requirement Notes
    Bash (or Git Bash on Windows) All DocTools scripts are pure Bash
    DocFX 2.x Must be on PATH. Verify with docfx --version
    .NET 9+ Required by DocFX for C# metadata extraction

    Installation

    Clone the DocTools repository:

    git clone <your-doctools-repo-url> /path/to/DocTools

    No additional installation is needed. Run commands directly via doctools.sh.

    Tip

    Add the DocTools directory to your PATH so you can call doctools.sh from anywhere.

    Initialize a Package

    Run init to scaffold a Documentation/ directory inside your package:

    doctools.sh init ./MyPackage --name "My Package" --title "My Package Docs"

    This creates:

    MyPackage/
└── Documentation/
    ├── doctools.config
    ├── toc.md
    ├── index.md
    └── Manual/
        ├── toc.md
        ├── getting-started.md
        └── Images/

    See Commands — init for all available flags.

    Build the Documentation

    Build all versions (in this case just the initial one):

    doctools.sh build ./MyPackage all

    Output goes to MyPackage/Documentation~/ by default:

    MyPackage/
└── Documentation~/
    ├── index.html      # Redirect to current version
    ├── versions.js     # Version list for the dropdown
    └── v0.1.0/
        ├── index.html
        ├── manual/
        └── styles/

    View the Output

    Open the generated site in your browser:

    start MyPackage/Documentation~/v0.1.0/index.html   # Windows
open MyPackage/Documentation~/v0.1.0/index.html     # macOS
xdg-open MyPackage/Documentation~/v0.1.0/index.html # Linux
