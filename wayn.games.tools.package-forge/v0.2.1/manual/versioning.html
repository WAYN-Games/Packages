    Versioning

    DocTools supports building documentation for multiple versions of a package simultaneously. Each version gets its own output directory, and the generated site includes a dropdown to switch between them.

    How It Works

    • ALL_VERSIONS in doctools.config lists every version, newest first.
    • CURRENT_VERSION identifies the version that receives C# metadata extraction during builds.
    • Historical versions are built from their own toc.md/index.md snapshots and shared manual content (or per-version overrides).
    • The build produces a versions.js file and a root index.html redirect pointing to the current version.

    Typical Workflow

    1. Start with a single version

    After doctools.sh init, you have one version (e.g., 0.1.0). Write your documentation and build normally.

    2. Snapshot before bumping

    Before adding a new version, snapshot the current TOC and index so they are preserved:

    doctools.sh snapshot ./MyPackage

    This copies Documentation/toc.md and Documentation/index.md into Documentation/v0.1.0/.

    3. Add the new version

    doctools.sh version ./MyPackage add 0.2.0

    This:

    • Sets CURRENT_VERSION to 0.2.0
    • Prepends 0.2.0 to ALL_VERSIONS
    • Creates Documentation/v0.2.0/ with copies of toc.md and index.md

    4. Update documentation for the new version

    Edit the root toc.md, index.md, and manual pages as needed. The v0.1.0/ directory retains the old TOC and index.

    5. Build all versions

    doctools.sh build ./MyPackage all

    Output:

    Documentation~/
├── index.html        # Redirects to v0.2.0
├── versions.js       # ["0.2.0", "0.1.0"]
├── v0.2.0/           # Built with metadata
└── v0.1.0/           # Built without metadata

    VERSION_CONTENT Overrides

    By default, all versions share the same Manual/ directory content. Use VERSION_CONTENT to customize what each version includes.

    declare -A VERSION_CONTENT
VERSION_CONTENT["0.1.0"]="manual_src|manual_exclude|toc_src"

    Each value has three pipe-delimited components:

    Component Default Purpose
    manual_src Manual Directory with manual markdown files
    manual_exclude Draft/**,Draft V2/** Comma-separated exclude patterns
    toc_src . Directory containing toc.md and index.md

    Example: Different TOC per version

    Store version-specific TOC and index in the version directory:

    VERSION_CONTENT["0.1.0"]="Manual||v0.1.0"

    This tells the build to use Documentation/v0.1.0/toc.md and Documentation/v0.1.0/index.md for version 0.1.0, while still pulling manual pages from the shared Manual/ directory.

    Example: Exclude draft content from a release

    VERSION_CONTENT["1.0.0"]="|Draft/**,Internal/**|"

    Version Switcher

    The generated site includes a version dropdown in the breadcrumb bar. It is populated at runtime from versions.js:

    window.DOC_VERSIONS = [{"version":"0.2.0"},{"version":"0.1.0"}];

    Selecting a different version navigates to ../v{version}/index.html.

    Directory Structure

    After several versions, the documentation directory looks like:

    Documentation/
├── doctools.config
├── toc.md              # Current version TOC
├── index.md            # Current version index
├── Manual/             # Shared manual pages
│   ├── toc.md
│   └── *.md
├── v0.1.0/             # Snapshot
│   ├── toc.md
│   └── index.md
└── v0.2.0/             # Snapshot
    ├── toc.md
    └── index.md
