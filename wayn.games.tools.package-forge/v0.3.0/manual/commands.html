    Commands

    DocTools provides four commands, all invoked through doctools.sh.

    doctools.sh <command> <package-dir> [arguments]

    init

    Scaffold a Documentation/ directory for a package.

    doctools.sh init <package-dir> [options]

    Options

    Flag Description Default
    --name NAME Package name Directory basename
    --title TITLE Documentation title Same as --name
    --version VER Initial version 0.1.0
    --footer FOOTER Footer HTML Empty

    Example

    doctools.sh init ./MyPackage --name "My Package" --title "My Package Docs" --version "1.0.0" --footer "&copy; 2025"

    Creates:

    MyPackage/Documentation/
├── doctools.config
├── toc.md
├── index.md
└── Manual/
    ├── toc.md
    ├── getting-started.md
    └── Images/

    build

    Build documentation for one or all versions.

    doctools.sh build <package-dir> [version|all]
    • all (default) — Builds every version in ALL_VERSIONS. The current version runs docfx metadata then docfx build; historical versions run docfx build only. Also generates versions.js and a root index.html redirect.
    • <version> — Builds a single version. If it matches CURRENT_VERSION, metadata extraction runs first.

    Example

    # Build all versions
doctools.sh build ./MyPackage all

# Build only version 0.2.0
doctools.sh build ./MyPackage 0.2.0

    Build Pipeline

    1. Load config — Sources Documentation/doctools.config, validates required fields, sets defaults.
    2. Metadata (current version only) — Runs docfx metadata to extract API docs from C# source into .tmp/api/.
    3. Generate docfx.json — Creates a temporary docfx.json with content entries, resource patterns, template paths, and global metadata.
    4. Build — Runs docfx build, outputting to {OUTPUT_DIR}/v{version}/.
    5. Output generation — Writes versions.js and (when building all) a root redirect index.html.

    Global Metadata

    Every page receives these metadata values:

    Key Value
    _appTitle PACKAGE_TITLE
    _appFooter PACKAGE_FOOTER
    _appName PACKAGE_NAME
    _appLogoPath LOGO_PATH
    _appFaviconPath FAVICON_PATH
    _disableContribution true
    _enableSearch true
    _disableNextArticle true
    _lang "en"
    _packageVersion Current build version

    version

    Manage documentation versions.

    doctools.sh version <package-dir> <action> [version]

    Actions

    list

    List all versions. The current version is marked with *.

    doctools.sh version ./MyPackage list

      0.2.0 *
  0.1.0

    add

    Add a new version. It becomes the current version and is prepended to ALL_VERSIONS. The current toc.md and index.md are copied into a v{version}/ directory.

    doctools.sh version ./MyPackage add 0.3.0

    remove

    Remove a version from the config. You cannot remove the current version — use set-current first.

    doctools.sh version ./MyPackage remove 0.1.0

    set-current

    Set which version is treated as current (receives metadata extraction on next build).

    doctools.sh version ./MyPackage set-current 0.2.0

    snapshot

    Copy the root toc.md and index.md into the current version's directory (v{CURRENT_VERSION}/). Use this to archive the table of contents before making changes for a new version.

    doctools.sh snapshot <package-dir>

    Example

    doctools.sh snapshot ./MyPackage

    Copies Documentation/toc.md and Documentation/index.md into Documentation/v{CURRENT_VERSION}/.
