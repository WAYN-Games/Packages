    编写自定义状态

    状态是角色行为的构建块。每个状态定义角色如何移动、对物理做出反应以及转换到其他状态。本指南涵盖状态接口、生命周期方法和常见模式。

    状态接口

    每个状态都是一个实现 ILocomotionState 的非托管结构体：

    [LocomotionState(typeof(MyCharacterStateMachine))]
internal partial struct MyState
{
    public void OnEnterState(ref MyContext ctx) { }

    public void OnIntegrateState(ref MyContext ctx) { }

    public void OnResolveStateCollisions(ref MyContext ctx) { }

    public void OnExitState(ref MyContext ctx) { }
}

    关键规则：

    • 所有生命周期方法都是带有默认接口实现（DIM 模式）的实例方法。它们是 public void，不是 internal static。ILocomotionState 接口由源代码生成器通过 partial 机制添加。
    • 转换由单独的转换结构体处理（见下方转换）——状态结构体不定义转换逻辑
    • 源代码生成器将调用分派到正确的状态结构体——运行时没有虚拟分派
    • 状态方法在已经 Burst 编译的 Job 内运行，因此状态结构体和方法上不需要显式 [BurstCompile]
    • 不能使用托管类型（字符串、类、数组）——此代码在 Burst Jobs 内运行

    生命周期方法

    OnEnterState

    当状态机转换进入此状态时调用一次。用于：

    • 初始化状态特定数据（重置计时器、计数器）
    • 设置初始速度
    • 清除接触信息（用于空中状态）
    public void OnEnterState(ref MyContext ctx)
{
    // 进入地面状态：取消所有垂直动量
    ctx.BaseContext.VerticalVelocity = float3.zero;
    ctx.BaseContext.VerticalDisplacement = float3.zero;
}

    OnIntegrateState

    在积分阶段每帧调用一次。这是您根据输入、重力和配置文件参数计算角色期望位移的地方。

    您的任务是在 BaseLocomotionContext 上填充 HorizontalDisplacement、VerticalDisplacement、HorizontalVelocity 和 VerticalVelocity。

    最简单的方式是计算加速度并调用 UpdateVelocitiesAndComputeDisplacements：

    public void OnIntegrateState(ref MyContext ctx)
{
    ref BaseLocomotionContext baseContext = ref ctx.BaseContext;

    // 计算重力
    float3 gravityAcceleration = baseContext.GravityDirection * baseContext.GravityStrength;

    // 从输入计算移动加速度
    float3 desiredVelocity = ctx.Input.MovementDirection * ctx.Profile.MaxSpeed;
    float accel = ctx.Profile.MaxSpeed / ctx.Profile.TimeToMaxSpeed;
    MathematicsHelpers.ComputeAccelerationToTargetVelocity(
        out float3 moveAcceleration,
        baseContext.HorizontalVelocity,
        desiredVelocity,
        baseContext.TimeStep,
        accel);

    // 应用两个加速度
    baseContext.UpdateVelocitiesAndComputeDisplacements(
        newHorizontalAcceleration: moveAcceleration,
        newVerticalAcceleration: gravityAcceleration);
}
    Important

    不要在此处添加平台速度。ComputeInheritedDisplacement 步骤在碰撞解析后自动处理。手动添加会导致双重应用。

    上下文拷贝模式

    由于 BaseLocomotionContext 是上下文接口上的值类型，修改它需要以下两种方式之一：

    方式 A：使用 ref 局部变量（当上下文暴露了后备字段时）：

    ref BaseLocomotionContext baseContext = ref ctx.BaseContext;
baseContext.HorizontalVelocity = float3.zero;
// 通过 ref 写入更改

    方式 B：拷贝、修改、写回（通过接口访问时）：

    BaseLocomotionContext baseContext = ctx.BaseContext;
baseContext.HorizontalVelocity = float3.zero;
ctx.BaseContext = baseContext;

    示例使用 ref 访问方式，因为上下文结构体除了接口属性外，还将 BaseContext 作为公共字段暴露。

    OnResolveStateCollisions

    每帧可能被调用多次（最多 MaxBounce 次迭代）。每次调用代表碰撞滑移算法的一次迭代。

    大多数情况下，您的实现只需委托给 baseContext.Move()：

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();
    ctx.BaseContext = baseContext;
}
    Note

    此处必须使用拷贝-修改-写回模式，因为 Move() 会在内部修改上下文的位置、位移和接触数据。

    对于高级用例，您可以添加自定义碰撞响应逻辑。例如，对角色碰到的动态物体施加冲量：

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();

    // 接触时将动态物体推开
    if (baseContext.IsInContact &&
        baseContext.LocomotionContact.ValueRO.BodyType == BodyType.Dynamic)
    {
        float3 pushVelocity = baseContext.HorizontalVelocity * 0.5f;
        baseContext.ApplyImpulse(
            ref baseContext,
            baseContext.LocomotionContact.ValueRO.BodyIndex,
            baseContext.LocomotionContact.ValueRO.Entity,
            /* impactorBodyIndex */ ...,
            baseContext.CharacterEntity,
            pushVelocity);
    }

    ctx.BaseContext = baseContext;
}

    OnExitState

    当状态机转换离开此状态时调用一次。用于清理状态特定数据：

    public void OnExitState(ref MyContext ctx)
{
    // 离开跳跃状态时重置跳跃计数器
    ctx.StateData.JumpData.JumpsCount = 0;
}

    转换

    转换定义角色何时以及如何在状态之间移动。本包支持两种模式。

    推荐：基于属性的转换

    将转换声明为带有 [LocomotionTransition] 的小型结构体。源代码生成器发现它们并自动处理分派、OnExitState 和 OnEnterState。

    [LocomotionTransition(typeof(IdleState), typeof(JumpingState))]
[EvaluateBefore(typeof(IdleToMoving))]
[EvaluateBefore(typeof(IdleToFalling))]
public struct IdleToJumping : IMyTransition
{
    public bool ShouldTransition(ref MyContext ctx)
        => ctx.BaseContext.IsGroundingContact && ctx.Input.JumpTrigger > 0;
}

    关键规则：

    1. 结构体名称遵循 {Source}To{Target} 约定
    2. [LocomotionTransition(typeof(FromState), typeof(ToState))] 声明连接哪些状态
    3. ShouldTransition 在转换应该触发时返回 true
    4. [EvaluateBefore] / [EvaluateAfter] 控制优先级——第一个匹配的转换获胜
    5. 源代码生成器自动调用源状态的 OnExitState 和目标状态的 OnEnterState

    查看示例中的 LocomotionTransitions.cs，其中包含 17 个转换的完整示例。

    基于属性的模式是唯一支持的方法。源代码生成器发现所有 [LocomotionTransition] 结构体，处理退出/进入调用，并通过 [EvaluateBefore]/[EvaluateAfter] 属性使转换优先级显式化。

    消费触发输入

    触发输入（如跳跃按键）在使用后必须被消费。否则，触发器会在每一帧都触发：

    static void ConsumeJumpTrigger(ref MyContext ctx)
{
    MyInput value = ctx.Input;
    value.JumpTrigger = 0;
    ctx.Input = value;
}

    在 OnIntegrateState 中使用触发值后调用此方法。由于 Input 是上下文上的值类型，因此需要使用拷贝-修改-写回模式。

    跨状态复用逻辑

    状态可以通过 internal static 辅助方法共享逻辑。例如，示例中的 JumpingState 复用了 FallingState.ComputeGravityAcceleration 计算重力，以及 MovingState.ComputeHorizontalMovementAcceleration 处理空中移动：

    public void OnIntegrateState(ref MyContext ctx)
{
    // 复用下落重力计算
    FallingState.ComputeGravityAcceleration(ref ctx, terminalVelocity, out float3 gravityAccel);

    // 复用移动水平加速度
    MovingState.ComputeHorizontalMovementAcceleration(
        ref ctx.BaseContext, ctx.Profile.Moving,
        ctx.Input.MovementDirection, ctx.Input.MovementMagnitude,
        out float3 moveAccel);

    ctx.BaseContext.UpdateVelocitiesAndComputeDisplacements(moveAccel, gravityAccel);
}

    这样可以避免重复物理逻辑，同时保持每个状态的独立性。

    常见状态模式

    地面状态（Idle / Moving）

    • OnEnterState：将垂直速度归零
    • OnIntegrateState：仅计算水平加速度
    • 转换：检查跳跃输入、失去地面接触、坡度超过最大值

    空中状态（Falling / Jumping）

    • OnEnterState：清除接触（LocomotionContact.None），可选择继承平台速度
    • OnIntegrateState：应用重力 + 可选空中控制
    • 转换：检查地面接触、墙壁接触

    滑行状态

    • OnIntegrateState：应用重力（角色沿斜坡滑下）
    • 转换：检查失去接触（转为下落）、地面接触低于最大坡度（转为移动/空闲）

    自定义状态创意

    • Wall Running（墙壁奔跑）：在高速跳向墙壁时进入；缓慢施加重力；允许蹬墙跳
    • Dashing（冲刺）：在触发时进入；施加瞬间水平速度；计时器结束后退出
    • Swimming（游泳）：在浸入水中时进入；用浮力替代重力；允许三维移动
    • Climbing（攀爬）：在梯子处进入；禁用重力；沿梯子轴移动

    调试状态

    如果您的状态行为不符合预期：

    1. 检查转换优先级顺序——可能有更早的转换先被触发
    2. 记录 baseContext.LocomotionContact.ValueRO.ContactType 以验证接触检测
    3. 验证转换期间 OnExitState 和 OnEnterState 是否被调用
    4. 确保触发输入在 ApplyInputsToCharacter 中累积并在 OnIntegrateState 中消费
    5. 使用调试可视化查看接触点、运动路径和冲量
