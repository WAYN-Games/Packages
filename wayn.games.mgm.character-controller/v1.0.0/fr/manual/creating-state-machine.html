    Créer une machine à états

    Ce guide explique comment créer une machine à états de locomotion complète de zéro, soit en utilisant l'outil de scaffolding de l'éditeur, soit manuellement.

    Utiliser l'outil de l'éditeur (recommandé)

    La façon la plus rapide de créer une machine à états est via le menu de l'éditeur Unity :

    1. Faites un clic droit dans la fenêtre Project à l'endroit où vous voulez le dossier de la machine à états
    2. Sélectionnez Assets > Create > WAYN Games > MGM Character Controller > StateMachine
    3. Tapez un nom pour votre machine à états (ex. Vehicle, Swimming, Flying)
    4. Appuyez sur Entrée

    L'outil génère tous les fichiers requis avec le nommage, les espaces de noms et le câblage appropriés. Vous pouvez immédiatement commencer à personnaliser le code généré.

    Comprendre ce qui est généré

    Lorsque vous créez une machine à états nommée MyCharacter, l'outil génère ces types :

    La struct ancre

    [LocomotionStateMachine]
public partial struct MyCharacterStateMachine { }

    Cette struct est le cœur de la machine à états. L'attribut [LocomotionStateMachine] est sans paramètre - tous les noms de types sont dérivés par convention du nom de la struct. Pour MyCharacterStateMachine, le préfixe MyCharacter produit : MyCharacterContext, MyCharacterProfile, MyCharacterInput, MyCharacterStateData et MyCharacterStates (l'enum généré).

    Les états sont des structs séparées annotées avec [LocomotionState], les reliant à la machine à états :

    [LocomotionState(typeof(MyCharacterStateMachine))]
public partial struct IdleState { }

    Le contexte

    La struct de contexte est générée par le source à partir de la struct ancre. Vous ne l'écrivez pas vous-même. La struct générée ressemble à :

    // Généré par le source - ne pas créer manuellement
public partial struct MyCharacterContext : ILocomotionContext<MyCharacterProfile, MyCharacterInput, MyCharacterStateData>
{
    public BaseLocomotionContext BaseContext { get; set; }
    public CharacterPhysicsProfile CharacterPhysicsProfile { get; set; }
    public MyCharacterProfile Profile { get; set; }
    public MyCharacterInput Input { get; set; }
    public MyCharacterStateData StateData { get; set; }
}

    Le contexte est un sac de données qui est passé par référence à travers chaque méthode du cycle de vie de l'état. Il encapsule :

    • BaseLocomotionContext - L'état physique central (position, vélocité, déplacement, contacts, gravité)
    • CharacterPhysicsProfile - La configuration capsule/physique bakée
    • Votre profil personnalisé, entrée et données d'état

    Parce que le contexte est partial, vous pouvez ajouter des propriétés personnalisées et des surcharges de méthodes (comme ComputeGroundSnappingDistance) dans votre propre déclaration partielle.

    Tip

    Si vous modifiez fréquemment les champs de BaseContext, déclarez-le comme champ public dans votre déclaration partielle plutôt que d'utiliser l'auto-propriété. Cela vous permet d'utiliser l'accès ref : ref BaseLocomotionContext baseContext = ref ctx.BaseContext;

    Le profil

    [Serializable]
public partial struct MyCharacterProfile
{
    // Ajoutez vos champs de configuration ici :
    // public float MaxSpeed;
    // public float JumpForce;
}

    L'interface ILocomotionProfile est ajoutée automatiquement par le générateur de source via le mécanisme partial. Les données de profil sont bakées dans un blob asset pendant la conversion d'entité. Elles sont immuables à l'exécution et partagées entre toutes les entités qui référencent le même asset de profil. Mettez tous les paramètres de gameplay réglables ici.

    L'entrée

    public partial struct MyCharacterInput
{
    // Ajoutez vos champs d'entrée ici :
    // public float3 Movement;
    // public float JumpTrigger;

    public void ApplyInputsToCharacter(in MyCharacterInput playerInputs)
    {
        // Copiez ou accumulez les valeurs d'entrée ici
    }
}

    Les interfaces ICharacterLocomotionInput<MyCharacterInput> et IComponentData sont ajoutées automatiquement par le générateur de source via le mécanisme partial. La struct d'entrée est un composant ECS. Elle reçoit les valeurs du PlayerInputAsset à la fréquence Update et les accumule pour la mise à jour fixe. La méthode ApplyInputsToCharacter est appelée à chaque frame Update par le système de lecture d'entrée généré.

    Warning

    Les entrées de type déclencheur (comme le saut) doivent être accumulées, pas écrasées. Si la fréquence Update est plus élevée que la fréquence de mise à jour fixe, un appui sur un déclencheur pourrait être manqué. Accumulez avec JumpTrigger += playerInputs.JumpTrigger et consommez-le dans votre logique d'état en le remettant à zéro.

    Le StateData

    public partial struct MyCharacterStateData
{
    // Ajoutez des données persistantes par état ici :
    // public int JumpCount;
    // public float StateTimer;
}

    L'interface IComponentData est ajoutée automatiquement par le générateur de source via le mécanisme partial. Les données d'état persistent entre les frames et les transitions d'état. Contrairement au contexte (qui est reconstruit à chaque frame à partir des composants), les données d'état sont stockées comme composant ECS et survivent entre les frames.

    L'asset de profil

    [CreateAssetMenu(...)]
public class MyCharacterProfileAsset : LocomotionProfileAsset<MyCharacterProfile>
{
    // Champs sérialisés correspondant à votre struct de profil

    public override ref MyCharacterProfile BuildBlobAsset(ref BlobBuilder blobBuilder)
    {
        ref MyCharacterProfile profile = ref blobBuilder.ConstructRoot<MyCharacterProfile>();
        // Copier les champs sérialisés vers le blob
        return ref profile;
    }
}

    Ce ScriptableObject est ce que vous assignez dans l'Inspector. Il sérialise vos données de profil et construit le blob asset pendant le baking.

    L'asset d'entrée

    [CreateAssetMenu(...)]
public class MyCharacterPlayerInputAsset : PlayerInputAsset<MyCharacterInput>
{
    public override void ReadInputsFromAsset(out MyCharacterInput inputs)
    {
        // Lire depuis Input System et remplir les entrées
    }

    public override void InitInputAsset(PlayerGameObjectInstance instance)
    {
        // Mettre en cache la caméra, le composant PlayerInput, etc.
    }
}

    L'asset d'entrée lit l'entrée matérielle et la convertit en votre struct d'entrée. C'est un ScriptableObject qui est instancié par joueur à l'exécution, de sorte que chaque joueur peut mettre en cache ses propres références de caméra et de périphérique d'entrée.

    Les composants d'authoring

    Deux MonoBehaviours d'authoring sont générés :

    1. Character Locomotor Authoring - Placé sur l'entité personnage dans la sub-scene. Référence l'asset de profil et configure le profil physique.
    2. Player Character Authoring - Placé sur l'entité joueur. Référence le personnage à contrôler et l'asset d'entrée.

    Le système de locomotion

    Le système de locomotion est entièrement généré par le source. Il implémente ILocomotionSystem<TStateMachine, TStates, TContext, TProfile, TInput, TStateData> et ISystem, planifiant le LocomotionJob à chaque mise à jour fixe. Vous n'avez jamais besoin d'écrire ce système vous-même.

    Pour personnaliser le comportement du système, surchargez les méthodes du cycle de vie DIM (OnLocomotionSystemCreate, OnLocomotionSystemUpdate, OnLocomotionSystemDestroy) via une déclaration d'interface partielle. Voir Personnaliser le système de locomotion pour les détails.

    Ce que le générateur de source produit

    En plus des fichiers que vous écrivez, le générateur de source Roslyn produit (à la compilation, invisible dans la fenêtre Project) :

    1. Composant de machine à états (MyCharacterStateMachine) - Une struct IComponentData avec un champ CurrentState et des méthodes de dispatch qui appellent l'implémentation d'état correcte en fonction de la valeur de l'enum
    2. Système de lecture d'entrée - Lit depuis le PlayerInputAsset à chaque frame Update et écrit dans le composant TInput
    3. Méthodes de cycle de vie ISystem - OnCreate, OnUpdate et OnDestroy générés sur votre struct de système de locomotion qui planifient le LocomotionJob et gèrent les type handles

    Vous n'avez jamais besoin d'écrire ou de maintenir ce code. Le générateur lit votre enum et le produit automatiquement.

    Ajouter un nouvel état

    Après la génération initiale, utilisez le menu de l'éditeur pour ajouter des états :

    1. Faites un clic droit dans la fenêtre Project
    2. Sélectionnez Assets > Create > WAYN Games > MGM Character Controller > State
    3. Nommez l'état (ex. Swimming)

    Cela génère un nouveau fichier de struct d'état annoté avec [LocomotionState(typeof(MyCharacterStateMachine))]. Le générateur de source ajoute automatiquement l'état à l'enum généré.

    Alternativement, créez la struct d'état manuellement et annotez-la avec [LocomotionState(typeof(YourStateMachine))]. Voir Écrire des états personnalisés pour le modèle de struct.

    Création manuelle

    Si vous préférez tout créer à la main (ou avez besoin de comprendre ce que font les outils), suivez cette checklist :

    1. Créez une partial struct ancre avec l'attribut [LocomotionStateMachine] (ex. MyCharacterStateMachine)
    2. Créez une partial struct pour les données de profil (nommée {Prefix}Profile)
    3. Créez une partial struct pour les données d'entrée avec ApplyInputsToCharacter (nommée {Prefix}Input)
    4. Créez une partial struct pour les données d'état (nommée {Prefix}StateData)
    5. Créez au moins une partial struct d'état avec [LocomotionState(typeof(YourStateMachine))]
    6. Créez un ScriptableObject LocomotionProfileAsset<TProfile>
    7. Créez un ScriptableObject PlayerInputAsset<TInput>
    8. Enregistrez le composant générique d'entrée : [assembly: RegisterGenericComponentType(typeof(LocomotionInputAsset<TInput>))]

    Le générateur de source produit la struct de contexte, l'enum d'état, le système de locomotion, les composants d'authoring et toutes les implémentations d'interface à partir de ces entrées.

    L'outil de l'éditeur fait tout cela en un clic, c'est pourquoi il est recommandé.
