    Premiers pas

    Ce guide vous accompagne dans la création de votre premier personnage de zéro en utilisant l'outil de scaffolding de l'éditeur.

    Si vous souhaitez simplement voir le package fonctionner rapidement, commencez plutôt par le Démarrage rapide (utilise l'exemple pré-construit).

    Prérequis

    • MGM Character Controller installé et compilant (Installation)
    • Familiarité de base avec les concepts Unity ECS (entités, composants, systèmes, baking). Si vous êtes nouveau avec ECS, consultez d'abord la documentation Entities de Unity.

    Configuration du projet

    Votre projet a besoin d'une sub-scene avec la physique activée. Le package repose sur Unity.Physics pour toute la détection de collision et n'utilise pas le système physique legacy.

    1. Créez ou ouvrez une scène
    2. Ajoutez une Sub Scene (clic droit dans la Hierarchy > New Sub Scene)
    3. Assurez-vous que la sub-scene a un composant Physics Step (présent automatiquement dans la plupart des configurations)
    4. Ajoutez des colliders statiques à votre environnement (sol, murs, etc.) en utilisant des composants Physics Shape

    Créer votre premier personnage

    La façon la plus rapide de commencer est d'utiliser l'outil de scaffolding intégré :

    Étape 1 : Générer la machine à états

    1. Dans la fenêtre Project, faites un clic droit sur le dossier où vous voulez créer votre machine à états
    2. Sélectionnez Assets > Create > WAYN Games > Locomotion > StateMachine
    3. Nommez le dossier (ex. MyCharacter)
    4. Appuyez sur Entrée

    L'outil génère une structure de dossiers minimale avec le fichier ancre. Tout le reste est généré par le code source à la compilation :

    MyCharacter/
├── Runtime/
│   ├── mycharacter.runtime.asmdef               # Définition d'assembly auto-générée
│   └── MyCharacterStateMachine.cs               # Struct ancre avec l'attribut [LocomotionStateMachine]
└── Authoring/
    └── mycharacter.authoring.asmdef             # Définition d'assembly auto-générée

    Après compilation, le générateur de source produit tous les types restants en mémoire (contexte, enum d'état, profil, entrée, données d'état, composants d'authoring, système de locomotion, système de lecture d'entrée). Vous ajoutez ensuite des états, transitions et structs de données via les sous-menus Assets > Create > WAYN Games > Locomotion.

    Étape 2 : Créer l'asset de profil

    1. Dans la fenêtre Project, faites un clic droit
    2. Sélectionnez l'élément de menu Create pour votre asset de profil de personnage (le chemin de menu est généré pour vous)
    3. Configurez la vitesse de déplacement, la force de saut, le multiplicateur de gravité et les autres paramètres dans l'Inspector

    Étape 3 : Créer l'asset d'entrée

    1. Faites un clic droit dans la fenêtre Project
    2. Sélectionnez l'élément de menu Create pour votre asset d'entrée du joueur
    3. Assignez vos Input Action References pour le mouvement et le saut

    Étape 4 : Configurer le prefab du personnage

    1. Créez un nouveau GameObject dans votre sub-scene
    2. Ajoutez le composant Character Locomotor Authoring généré
    3. Assignez votre Profile Asset
    4. Configurez le Physics Profile (hauteur de la capsule, rayon, masse, pente maximale, etc.)

    Étape 5 : Configurer le prefab du joueur

    Le Player est un concept séparé du Character. Les joueurs possèdent les entrées et contrôlent un ou plusieurs personnages.

    1. Créez un GameObject dans la sub-scene (il est converti en entité Player)
    2. Ajoutez le composant Player Character Authoring généré
    3. Assignez le Controlled Character (le composant d'authoring du Locomotor), l'Input Asset et le Player GO Prefab (un prefab GameObject classique contenant les composants caméra et entrée, enregistré en dehors de la sub-scene)

    Étape 6 : Entrer en mode Play

    Appuyez sur Play. Votre personnage devrait répondre aux entrées et interagir avec l'environnement physique. Si le personnage tombe à travers le sol, vérifiez que :

    • Votre sol a un collider physique
    • Les couches de collision du personnage (BelongsTo et CollidesWith) sont configurées correctement
    • La sub-scene est chargée

    Importer l'exemple

    Le package inclut un SinglePlayerSample qui démontre une configuration complète avec cinq états (Idle, Moving, Jumping, Falling, Sliding), un rig de caméra, des plateformes mobiles et la visualisation de débogage.

    Pour étudier l'exemple :

    1. Naviguez jusqu'à Assets/MGM Locomotion/Samples/SinglePlayerSample/
    2. Ouvrez la scène dans Scenes/
    3. Examinez les scripts dans Scripts/SinglePlayerStateMachine/ pour une implémentation de référence complète
    Tip

    L'exemple est la meilleure ressource pour comprendre comment toutes les pièces se connectent. Lisez-le en parallèle de cette documentation.
