    Recette : Ajouter un nouvel état

    Ajoutez un nouveau comportement de mouvement (par exemple, le Dash) à une machine à états existante.

    Quand l'utiliser

    Vous souhaitez un nouveau mode de déplacement distinct des états existants (physique différente, gestion des entrées différente, transitions différentes).

    Prérequis

    • Une machine à états existante créée avec l'outil de scaffolding ou manuellement
    • Être familier avec Écrire des états personnalisés

    Étapes

    1. Générer le fichier d'état

    1. Faites un clic droit dans la fenêtre Project
    2. Sélectionnez l'élément de menu généré : Assets > Create > WAYN Games > Locomotion > YourStateMachine > State
    3. Nommez-le (par exemple, Dashing)
    4. Appuyez sur Entrée

    Cela crée un nouveau fichier DashingState.cs et ajoute l'entrée d'énumération avec l'attribut [LocomotionState].

    2. Ajouter des données de profil (si nécessaire)

    Si le nouvel état nécessite des paramètres ajustables, ajoutez un sous-profil :

    [Serializable]
public struct DashProfile
{
    public float DashSpeed;
    public float DashDuration;
    public float DashCooldown;
}

    Ajoutez-le à votre struct de profil principal :

    public partial struct MyProfile : ILocomotionProfile
{
    public MovingProfile Moving;
    public FallingProfile Falling;
    public JumpingProfile Jumping;
    public DashProfile Dash; // New
}

    Mettez à jour votre méthode LocomotionProfileAsset.BuildBlobAsset pour copier les nouveaux champs.

    3. Ajouter des données d'état (si nécessaire)

    Si l'état doit suivre des données entre les frames (minuteries, compteurs), ajoutez des champs à votre struct de données d'état :

    public partial struct MyStateData : IComponentData
{
    public JumpData JumpData;
    public float DashTimer;      // New
    public float DashCooldown;   // New
}

    4. Implémenter la logique de l'état

    [LocomotionState(typeof(MyStateMachine))]
internal partial struct DashingState
{
    public void OnEnterState(ref MyContext ctx)
    {
        // Set initial dash velocity in the current facing direction
        ctx.BaseContext.HorizontalVelocity =
            math.forward(ctx.BaseContext.CharacterPosition.Rotation) * ctx.Profile.Dash.DashSpeed;
        ctx.BaseContext.VerticalVelocity = float3.zero;
        ctx.StateData.DashTimer = ctx.Profile.Dash.DashDuration;
    }

    public void OnIntegrateState(ref MyContext ctx)
    {
        ref BaseLocomotionContext baseContext = ref ctx.BaseContext;
        // Maintain dash velocity, no acceleration
        baseContext.UpdateVelocitiesAndComputeDisplacements();
        ctx.StateData.DashTimer -= baseContext.TimeStep;
    }

    public void OnExitState(ref MyContext ctx)
    {
        ctx.StateData.DashCooldown = ctx.Profile.Dash.DashCooldown;
    }
}

    5. Ajouter des transitions

    Créez des structs de transition pour entrer et sortir du nouvel état :

    [LocomotionTransition]
[EvaluateBefore(typeof(MovingToJumping))] // Higher priority than jump
public struct MovingToDashing : ILocomotionTransition<MovingState, DashingState>
{
    public bool ShouldTransition(ref MyContext ctx)
        => ctx.Input.DashTrigger > 0 && ctx.StateData.DashCooldown <= 0;
}

[LocomotionTransition]
public struct DashingToMoving : ILocomotionTransition<DashingState, MovingState>
{
    public bool ShouldTransition(ref MyContext ctx)
        => ctx.StateData.DashTimer <= 0;
}

[LocomotionTransition]
[EvaluateBefore(typeof(DashingToMoving))]
public struct DashingToFalling : ILocomotionTransition<DashingState, FallingState>
{
    public bool ShouldTransition(ref MyContext ctx)
        => !ctx.BaseContext.IsInContact;
}

    6. Ajouter une entrée (si nécessaire)

    Si l'état nécessite une nouvelle action d'entrée (par exemple, un déclencheur de dash), ajoutez-la à votre struct d'entrée et à votre PlayerInputAsset. Voir Système d'entrée.

    Résultat attendu

    Le personnage effectue un dash vers l'avant à grande vitesse lorsque le déclencheur de dash est pressé, maintient cette vélocité pendant la durée configurée, puis revient à l'état de déplacement (ou de chute s'il est en l'air).

    Erreurs courantes

    • Utiliser des types managés (chaînes, classes, tableaux) dans la logique d'état - les états s'exécutent dans des jobs compilés par Burst
    • Ne pas consommer les entrées de déclencheur (le dash se déclenche à chaque frame)
    • Ne pas initialiser newState = default au début des helpers de transition (avertissement de compilation)
