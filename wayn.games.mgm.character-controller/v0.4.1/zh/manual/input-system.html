    输入系统

    MGM Character Controller 将输入读取与输入消费分离。此设计支持多种输入源（键盘、手柄、AI、网络），处理 Update 和 FixedUpdate 之间的帧率差异，并与 Unity 的新 Input System 良好配合。

    架构

    输入流经三个层：

    flowchart LR
    subgraph L1["1. Read"]
        A["<b>PlayerInputAsset</b><br/>MonoBehaviour · ScriptableObject<br/>Reads hardware input<br/><i>Update rate</i>"]
    end

    subgraph L2["2. Accumulate"]
        B["<b>ICharacterLocomotionInput</b><br/>ECS component<br/>ApplyInputsToCharacter()<br/><i>Update rate</i>"]
    end

    subgraph L3["3. Consume"]
        C["<b>State logic</b><br/>Burst job<br/>Reads TInput on context<br/><i>FixedUpdate rate</i>"]
    end

    A -- "ReadInputsFromAsset(out TInput)" --> B
    B -- "generated InputReaderSystem" --> C

    为什么需要三个层？

    帧率不匹配：Unity 的 Update 循环以可变速率运行（例如 60-144 fps），但 FixedUpdate 以固定速率运行（例如 50 fps）。如果玩家在没有 FixedUpdate 运行的帧中按下跳跃，该按压不能丢失。累积层确保触发输入能保留到下一个固定步。

    Burst 兼容性：PlayerInputAsset 是一个 ScriptableObject，它使用 MonoBehaviour API（Camera、PlayerInput、Input System）。这些无法在 Burst 作业内运行。这种分离允许托管代码层读取输入，而 Burst 编译层消费输入。

    多输入源：通过将输入抽象到 ICharacterLocomotionInput 之后，你可以更换输入源而无需更改任何状态逻辑。键盘玩家、手柄玩家和 AI 代理都写入同一个 TInput 组件。

    PlayerInputAsset

    PlayerInputAsset<TInput> 是一个抽象 ScriptableObject，你需要子类化它以定义硬件输入如何映射到你的输入结构体。

    [CreateAssetMenu(menuName = "My Game/Player Input Asset")]
public class MyPlayerInputAsset : PlayerInputAsset<MyInput>
{
    public InputActionReference MovementAction;
    public InputActionReference JumpAction;

    Camera _camera;
    PlayerInput _playerInput;

    public override void ReadInputsFromAsset(out MyInput inputs)
    {
        // Read raw input
        var moveRaw = _playerInput.actions[MovementAction.name].ReadValue<Vector2>();

        // Convert to world-space relative to camera
        Transform cam = _camera.transform;
        float3 worldMove = cam.forward * moveRaw.y + cam.right * moveRaw.x;
        worldMove -= math.projectsafe(worldMove, math.up()); // Remove vertical component
        inputs.Movement = math.normalizesafe(worldMove);

        // Read trigger
        inputs.JumpTrigger = _playerInput.actions[JumpAction.name].WasPerformedThisFrame() ? 1f : 0f;
    }

    public override void InitInputAsset(PlayerGameObjectInstance instance)
    {
        var playerGo = instance.Instance.Value;
        _camera = playerGo.GetComponent<PlayerGoAuthoring>().Camera;
        _playerInput = playerGo.GetComponent<PlayerInput>();
        _playerInput.ActivateInput();
    }
}

    要点

    • ReadInputsFromAsset 在每个 Update 帧调用（不是 FixedUpdate）。每次调用返回一个新的 TInput。
    • InitInputAsset 在玩家实例化时调用一次。使用它来缓存对相机、PlayerInput 和任何其他托管对象的引用。
    • 该资产在运行时按玩家实例化（ScriptableObject.Instantiate），因此每个玩家都有自己的缓存引用。同一台机器上的多个玩家各自拥有自己的副本。

    相机相对输入

    大多数 3D 游戏将 2D 摇杆/键盘输入转换为相对于相机的世界空间移动。模式是：

    1. 读取 2D 输入向量
    2. 乘以相机的 forward 和 right 向量
    3. 投影到水平面上（移除重力轴分量）
    4. 归一化

    这确保"向上推摇杆"始终意味着"远离相机移动"，无论相机角度如何。

    ICharacterLocomotionInput

    你的输入结构体必须实现此接口：

    public struct MyInput : ICharacterLocomotionInput<MyInput>
{
    public float3 Movement;
    public float JumpTrigger;

    public void ApplyInputsToCharacter(in MyInput playerInputs)
    {
        // Continuous inputs: overwrite with latest value
        Movement = playerInputs.Movement;

        // Trigger inputs: ACCUMULATE, don't overwrite
        JumpTrigger += playerInputs.JumpTrigger;
    }
}

    输入累积

    关键区别在于持续输入和触发输入：

    类型 示例 行为 原因
    持续 移动方向、瞄准角度 每帧覆盖 只有最新值有意义
    触发 跳跃按下、开火按钮 跨帧累积 在没有 FixedUpdate 运行的帧中的按压不能丢失

    如果你覆盖 JumpTrigger 而不是累积它，且玩家在没有 FixedUpdate 运行的帧中按下跳跃，该按压会被静默丢弃。

    消费触发器

    在你的状态的 OnIntegrateState 中，读取触发值后将其重置为零：

    internal static void OnIntegrateState(ref MyContext ctx)
{
    if (ctx.Input.JumpTrigger > 0)
    {
        // Apply jump
        Jump(ref ctx);
    }
    // Consume the trigger
    MyInput input = ctx.Input;
    input.JumpTrigger = 0;
    ctx.Input = input;
}

    生成的输入读取器系统

    源代码生成器创建一个输入读取器系统，它：

    1. 在每个 Update 帧运行（不是 FixedUpdate）
    2. 对每个玩家，调用 PlayerInputAsset.ReadInputsFromAsset 获取原始输入
    3. 对该玩家控制的每个角色，调用 ICharacterLocomotionInput.ApplyInputsToCharacter 写入 ECS 组件

    你永远不需要编写或修改此系统。它由你的 [LocomotionStateMachine] 注解自动生成。

    AI 和网络输入

    由于输入只是一个 ECS 组件，你可以从任何来源驱动角色：

    AI 输入

    创建一个系统直接写入 TInput 组件，无需通过 PlayerInputAsset：

    [BurstCompile]
partial struct AIInputSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        foreach (var (input, transform) in
            SystemAPI.Query<RefRW<MyInput>, RefRO<LocalTransform>>()
                .WithAll<AIControlled>())
        {
            // AI decision-making logic
            input.ValueRW.Movement = ComputeAIMovement(transform.ValueRO);
            input.ValueRW.JumpTrigger = ShouldJump() ? 1f : 0f;
        }
    }
}

    网络输入

    对于联网游戏，客户端将输入发送到服务器，服务器将其写入 TInput 组件。运动作业在客户端和服务器上运行完全相同，产生确定性结果（假设相同的输入和物理状态）。

    Cinemachine 集成

    示例的 LocomotionPlayerInputAsset 演示了 Cinemachine 相机设置：

    1. 在玩家预制体上找到所有 CinemachineVirtualCameraBase 组件
    2. 将它们的 Follow 和 LookAt 目标设置为角色的相机目标变换
    3. 在 CinemachineBrain 上设置相机覆盖以在玩家的虚拟相机之间混合
    4. 为每个玩家的相机控制分配 CinemachineInputAxisController 的玩家索引

    这确保分屏设置中的每个玩家都有自己的相机管线。
