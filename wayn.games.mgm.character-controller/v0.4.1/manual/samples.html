    Samples

    This page lists all samples included with the MGM Character Controller and provides a recommended study order.

    SinglePlayerSample
    Teaches Complete state machine with five states, transitions, input handling, camera setup, and profile configuration
    Difficulty Beginner to intermediate
    Setup time 5 minutes (see Quickstart)
    Dependencies Unity Input System, Cinemachine

    What It Contains

    The sample provides a full reference implementation of a third-person character controller:

    • 5 states: Idle, Moving, Jumping, Falling, Sliding
    • 18 transitions: All valid state-to-state transitions with priority ordering
    • Input handling: Camera-relative movement with jump trigger accumulation
    • Profile: Tunable speed, acceleration, jump force, gravity, and terminal velocity
    • Prefabs: Character (Locomotor + Model) and Player (Camera + Input)
    • Assets: Pre-configured profile and input ScriptableObjects

    Key Files - Study Order

    Read the sample code in this order to build the mental model:

    Order File What It Teaches
    1 Runtime/LocomotionStateMachine.cs The anchor struct - [LocomotionStateMachine] attribute triggers source generation
    2 Runtime/Data/LocomotionInput.cs Input struct with continuous (Movement) and trigger (JumpTrigger) fields, plus accumulation in ApplyInputsToCharacter
    3 Runtime/Data/LocomotionProfile.cs Profile struct composed of sub-profiles (MovingProfile, FallingProfile, JumpingProfile)
    4 Runtime/Data/LocomotionStateData.cs Per-state persistent data (JumpData with jump count)
    5 Runtime/States/IdleState.cs Simplest state - zeros velocity on enter, handles platform rotation
    6 Runtime/States/MovingState.cs Horizontal acceleration from input, character rotation toward movement direction
    7 Runtime/States/FallingState.cs Gravity application with terminal velocity, reuses MovingState's horizontal acceleration
    8 Runtime/States/JumpingState.cs Jump force application, trigger consumption, multi-jump support via StateData
    9 Runtime/States/SlidingState.cs Gravity-driven sliding on steep slopes
    10 Runtime/Transitions/LocomotionTransitions.cs All transitions as small structs with [LocomotionTransition] and [EvaluateBefore] for priority ordering
    11 Runtime/Data/LocomotionContextExtensions.cs Custom ground snapping distance that accounts for slope angle and platform velocity
    12 Runtime/Data/LocomotionPlayerInputAsset.cs Camera-relative input reading and Cinemachine setup

    Things to Try

    After running the sample, try these modifications to deepen your understanding:

    1. Change movement speed - Open the SinglePlayerProfileAsset and modify Moving > MaxSpeed. Notice how the profile is baked, so you need to re-enter Play mode.
    2. Enable double jump - In the profile, set Jumping > MaxJumps to 2. The JumpingState already supports multi-jump via StateData.JumpData.JumpsCount.
    3. Disable air control - In JumpingState.OnIntegrateState, remove the horizontal acceleration block. The character will maintain its launch velocity with no air steering.
    4. Add debug visualization - Select the character entity in the sub-scene and add a LocomotionDebugConfigAuthoring component. Enter Play mode and observe contact points and movement paths in the Scene view.
