    Moving Platforms

    The MGM Character Controller has built-in support for moving and rotating platforms. Characters standing on platforms automatically inherit their velocity and rotation without any special logic in your state code.

    How It Works

    Moving platforms are kinematic physics bodies tagged with the Platform component. The locomotion system detects when a character is grounded on a platform and automatically computes the velocity and rotation the character should inherit.

    The process happens in two steps across two frames:

    Frame N: Contact Detection

    During the SnapToGround step, the system detects that the character is standing on a Platform entity. It records the platform's body index and entity reference in the LocomotionContact component.

    Frame N: ComputeInheritedDisplacement

    At the end of the locomotion job, the ComputeInheritedDisplacement step reads the platform's current velocity from the physics world and decomposes it into:

    • PlatformHorizontalVelocity - The platform's velocity projected onto the horizontal plane
    • PlatformVerticalVelocity - The platform's velocity along the gravity axis
    • InheritedRotation - The rotational change the character should inherit

    Frame N+1: Application

    In the next frame, the character's position is adjusted by the platform's displacement before state logic runs. The inherited rotation is available to your OnIntegrateState for use in rotation logic (e.g., the Idle state's platform spin behavior).

    Important

    You should never add platform velocity manually in OnIntegrateState. The system handles it automatically. Adding it yourself would result in double-application, making the character move twice as fast on platforms.

    Platform Components

    Platform (Tag)

    A zero-size tag component marking an entity as a moving platform. Applied during baking by PlatformAuthoring.

    PlatformMover

    Stores linear velocity data for the platform. Updated by the PlatformMovementSystem.

    Rotor

    Stores angular velocity for rotating platforms. The system applies the rotation each fixed update.

    Track

    Constrains platform motion along a predefined path. The platform oscillates between waypoints.

    Setting Up a Moving Platform

    1. Create the platform GameObject (in sub-scene)

    1. Create a GameObject with your platform mesh
    2. Add a physics collider (Box, Mesh, etc.)
    3. Set the collider as Kinematic (not Dynamic, not Static)

    2. Add platform authoring

    Add a PlatformAuthoring component. This tags the entity with the Platform component during baking.

    3. Add movement behavior

    Add a MovingPlatformAuthoring component and configure:

    • Linear Velocity: Direction and speed of movement
    • Angular Velocity: Rotation speed around each axis
    • Track (optional): Define waypoints for the platform to follow

    4. Collision layers

    Ensure the platform's collision layer is included in your character's CollidesWith mask. By default, both use all layers, so this works out of the box.

    PlatformMovementSystem

    The PlatformMovementSystem runs in the FixedStepSimulationSystemGroup and updates platform positions and rotations based on their PlatformMover and Rotor components. It writes to the PhysicsVelocity component, which Unity.Physics uses to move the kinematic body.

    Character Behavior on Platforms

    Standing Still

    When idle on a platform, the character moves with the platform. If the platform rotates, the character's rotation is updated by the InheritedRotation (see the sample's IdleState.OnIntegrateState for the spin/roll/slide interpolation logic).

    Moving on a Platform

    When walking on a moving platform, the character's own velocity is additive with the platform's velocity. The character moves relative to the platform surface.

    Jumping off a Platform

    When the character jumps, the JumpingState.OnEnterState adds PlatformHorizontalVelocity to the character's horizontal velocity:

    ctx.BaseContext.HorizontalVelocity += ctx.BaseContext.PlatformHorizontalVelocity;

    This preserves the platform's momentum, so jumping off a fast-moving platform launches the character in the direction the platform was moving. Without this, the character would appear to "snap back" to a standstill mid-air.

    Platform Push (Overlap Resolution)

    If a platform moves into the character (e.g., a rising elevator ceiling), the SolveOverlaps step pushes the character out. If the character is trapped between two surfaces (e.g., elevator ceiling and floor), the overlap state transitions through Push -> Pinch -> Crush.

    Rotating Platform Interaction

    The SpinOrRollAlongWall parameter on CharacterPhysicsProfile controls how the character responds to being on a rotating platform while touching a wall:

    SpinOrRollAlongWall = -1:  Character spins with the platform, slides along wall
SpinOrRollAlongWall =  0:  Character slides along wall without rotating
SpinOrRollAlongWall =  1:  Character "rolls" on the wall surface

    This is interpolated based on the contact geometry and the relationship between the gravity direction, contact normal, and platform velocity.

    Custom Platform Behaviors

    You can create custom platform movement by writing your own system that modifies the PhysicsVelocity component on platform entities. As long as the entity has the Platform tag, the locomotion system will automatically handle character inheritance.

    For example, a platform that follows a spline:

    [BurstCompile]
partial struct SplinePlatformSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        float dt = SystemAPI.Time.fixedDeltaTime;
        foreach (var (velocity, spline, transform) in
            SystemAPI.Query<RefRW<PhysicsVelocity>, RefRW<SplineFollower>, RefRO<LocalTransform>>()
                .WithAll<Platform>())
        {
            float3 targetPos = EvaluateSpline(spline.ValueRO, SystemAPI.Time.ElapsedTime);
            velocity.ValueRW.Linear = (targetPos - transform.ValueRO.Position) / dt;
        }
    }
}

    Gravity Wells

    The sample includes a GravityWellAuthoring component that modifies the LocomotionGravity component on nearby characters. This is not directly related to platforms but demonstrates how you can modify per-character physics at runtime.
