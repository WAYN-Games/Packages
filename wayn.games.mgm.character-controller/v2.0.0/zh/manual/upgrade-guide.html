    升级指南

    本页记录版本之间的重大更改和迁移步骤。

    0.4.x 到 0.5.0

    发布日期：2026-03-24

    重大更改

    包重命名

    包标识符从 com.wayngamesassets.mgm.charactercontroller 更改为 wayn.games.mgm.character-controller。

    迁移：在 Unity 项目的 Packages/manifest.json 中，更新依赖名称：

    // 之前
"com.wayngamesassets.mgm.charactercontroller": "..."

// 之后
"wayn.games.mgm.character-controller": "..."

    命名空间重命名

    所有命名空间从 WAYN.Games.MGM.Locomotion.* 更改为 WAYN.Games.MGM.CharacterController.*。

    旧命名空间 新命名空间
    WAYN.Games.MGM.Locomotion.Runtime.StateMachine WAYN.Games.MGM.CharacterController.Runtime.StateMachine
    WAYN.Games.MGM.Locomotion.Runtime.Components WAYN.Games.MGM.CharacterController.Runtime.Components
    WAYN.Games.MGM.Locomotion.Runtime.Physics WAYN.Games.MGM.CharacterController.Runtime.Physics
    WAYN.Games.MGM.Locomotion.Authoring WAYN.Games.MGM.CharacterController.Authoring
    WAYN.Games.MGM.Locomotion.Hybrid.* WAYN.Games.MGM.CharacterController.Hybrid.*

    迁移：在所有 .cs 文件中查找并替换：

    WAYN.Games.MGM.Locomotion → WAYN.Games.MGM.CharacterController

    程序集定义重命名

    所有程序集名称已更改。如果您通过名称（而非 GUID）引用它们，请更新引用：

    旧程序集名称 新程序集名称
    Wayn.Games.Locomotion.Runtime WAYN.Games.MGM.CharacterController.Runtime
    Wayn.Games.Locomotion.Editor WAYN.Games.MGM.CharacterController.Editor
    Wayn.Games.Locomotion.Hybrid WAYN.Games.MGM.CharacterController.Hybrid

    迁移：如果您的 .asmdef 文件通过 GUID 引用包程序集，则无需更改。如果通过名称引用，请更新为新名称。

    编辑器菜单路径更改

    Unity 编辑器菜单从 Assets > Create > WAYN Games > Locomotion 移至 Assets > Create > WAYN Games > MGM Character Controller。

    如何验证升级

    1. 使用新包名更新 manifest.json
    2. 在所有脚本中查找并替换旧命名空间
    3. 编译项目 — 修复所有剩余的类型引用
    4. 进入 Play 模式，验证角色移动是否正常工作

    0.3.0 到 0.4.0

    发布日期：2026-03-22

    重大更改

    源代码生成器重写

    源代码生成器已重写。ILocomotionStateMachineDefinition 已被移除，替换为基于属性的发现。

    之前 (0.3.0)：

    public struct MyStateMachineDefinition : ILocomotionStateMachineDefinition { ... }

    之后 (0.4.0)：

    [LocomotionStateMachine]
public partial struct MyStateMachine { }

    迁移：删除旧的定义结构体。创建一个带有 [LocomotionStateMachine] 属性的新结构体。源代码生成器从 [LocomotionState] 和 [LocomotionTransition] 属性发现状态和转换。

    DIM 生命周期默认值

    源代码生成器现在为默认接口方法生命周期钩子生成显式结构体转发方法。如果您之前依赖于 DIM 默认值的隐式调用，行为应保持不变。如果您在 partial 结构体上覆盖了 DIM，请验证您的覆盖仍然可以编译。

    编辑器菜单重组

    创建菜单、脚本模板和辅助工具移到了 Editor/CreationMenus/ 子文件夹中。如果您有引用旧路径的脚本，请更新引用。

    Locomotion 脚本脚手架

    新的 CreateLocomotionScript 菜单和 LocomotionScriptCreationMenus 替代了旧的脚手架工具。生成的文件夹结构和文件命名已更改：

    • 所有生成的文件现在以状态机名称为前缀，以避免多个状态机之间的冲突
    • 程序集定义会自动生成，并包含正确的命名空间和 GUID 引用

    迁移：如果您使用了旧的脚手架工具，已生成的文件仍然有效。由更新后的工具生成的新文件将使用新的命名规范。

    如何验证升级

    1. 在 Unity 6000.3+ 中打开项目
    2. 等待编译完成
    3. 检查 Console 中的错误 - 修复所有缺失的类型引用
    4. 使用现有场景进入 Play 模式
    5. 验证角色移动是否正常工作

    0.2.0 到 0.3.0

    发布日期：2026-03-18

    重大更改

    ILocomotionState 使用实例方法 DIM 模式

    状态生命周期方法从 internal static 更改为带有默认接口实现的实例方法。

    之前 (0.2.0)：

    internal static void OnEnterState(ref MyContext ctx) { }

    之后 (0.3.0)：

    public void OnEnterState(ref MyContext ctx) { }

    迁移：将所有状态生命周期方法从 internal static 更改为 public 实例方法。

    CharacterPhysicsProfile 移至 ILocomotionContext

    CharacterPhysicsProfile 不再是 ILocomotionProfile 的一部分。它现在是 ILocomotionContext 上的属性。

    之前：ctx.Profile.PhysicsProfileValue 之后：ctx.CharacterPhysicsProfile

    迁移：将物理 profile 的访问从 ctx.Profile 移至 ctx.CharacterPhysicsProfile。从您的 profile 结构体中移除 PhysicsProfileValue。

    基础编写类重命名

    旧名称 新名称
    BaseCharacterAuthoring BaseCharacterLocomotorAuthoring
    BaseCharacterBaker BaseCharacterLocomotorBaker

    迁移：在您的 authoring 脚本中查找并替换旧名称。

    平台速度继承

    基于路径跟踪的平台系统被替换为通过 ComputeInheritedDisplacement 实现的基于 MotionVelocity 的方法。

    迁移：如果您使用了旧的 PlatformPath 组件，请将其移除。平台继承现在对角色站立的任何动态刚体自动工作。通过 ctx.BaseContext.PlatformHorizontalVelocity 和 ctx.BaseContext.PlatformVerticalVelocity 访问平台数据。

    转换评估时机

    EvaluateTransitions 现在在执行循环结束时（所有物理计算之后）运行一次，而不是在每次弹跳迭代后运行。

    迁移：如果您的转换依赖于帧中间的接触数据，请调整它们以使用最终的物理后接触状态。

    如何验证

    1. 编译项目 - 修复重命名的类型和更改的方法签名
    2. 进入 Play 模式，测试所有状态和转换
    3. 测试移动平台 - 验证角色能正确站在平台上
    4. 启用调试可视化以确认接触检测仍然正常工作

    0.1.0 到 0.2.0

    发布日期：2025-10-05

    更改

    • 添加了状态机系统的源代码生成器
    • 添加了状态机创建菜单
    • 重构了系统以支持用户定义的状态机

    迁移：0.1.0 API 是内部/预览版。使用新的创建菜单（Assets > Create > WAYN Games > MGM Character Controller > StateMachine）重新创建您的状态机。
