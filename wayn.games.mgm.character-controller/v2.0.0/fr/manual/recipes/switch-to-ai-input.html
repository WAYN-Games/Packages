    Recette : Passer un personnage en input IA

    Pilotez un personnage avec un systeme d'IA au lieu de l'input joueur.

    Quand l'utiliser

    Vous souhaitez des personnages PNJ qui utilisent la meme state machine et la meme logique de deplacement que les joueurs, mais qui sont controles par une prise de decision IA au lieu d'un peripherique d'entree.

    Prerequis

    • Un personnage fonctionnel avec une state machine
    • Une connaissance du Systeme d'input

    Etapes

    1. Creer un composant tag

    public struct AIControlled : IComponentData { }

    Ajoutez-le aux entites de personnages controlees par l'IA pendant le baking (via un composant d'authoring ou un systeme).

    2. Creer un systeme d'input IA

    Ecrivez un systeme qui ecrit directement dans le composant ECS d'input, en contournant PlayerInputAsset :

    [BurstCompile]
[UpdateInGroup(typeof(SimulationSystemGroup))]
partial struct AIInputSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        foreach (var (input, transform) in
            SystemAPI.Query<RefRW<MyInput>, RefRO<LocalTransform>>()
                .WithAll<AIControlled>())
        {
            // Exemple : se deplacer vers une position cible
            float3 targetDir = math.normalizesafe(
                GetTargetPosition() - transform.ValueRO.Position);

            input.ValueRW.Movement = targetDir;
            input.ValueRW.JumpTrigger = ShouldJump() ? 1f : 0f;
        }
    }
}

    3. Ne pas creer d'entite Player pour les personnages IA

    Les personnages IA n'ont pas besoin d'une entite Player, d'un prefab player ou d'un PlayerInputAsset. Ils ont uniquement besoin de l'entite Locomotor avec les composants de la state machine et le tag AIControlled.

    Pendant le baking, ajoutez le composant d'input directement sans le pipeline d'input joueur :

    public class AICharacterAuthoring : MonoBehaviour
{
    // Le baking ajoute le composant MyInput + le tag AIControlled
}

    4. (Optionnel) State machine partagee

    Le personnage IA utilise exactement la meme state machine, les memes etats, transitions, profil et physique que les personnages joueurs. Aucune modification de code n'est necessaire dans la state machine : elle ne lit que le composant TInput, qui est ecrit par votre systeme IA au lieu du pipeline d'input joueur.

    Resultat attendu

    Les personnages IA se deplacent, sautent et transitionnent entre les etats de maniere identique aux personnages joueurs. La seule difference est la source des donnees d'input.

    Erreurs courantes

    • Oublier d'ajouter le composant d'input a l'entite IA : le systeme de locomotion ne la traitera pas
    • Ecrire l'input a chaque frame sans accumuler les declencheurs : les declencheurs de saut peuvent etre manques entre les mises a jour fixes (utilisez le meme pattern d'accumulation que l'input joueur)
