    Integration : Animation

    Les personnages pilotes par le MGM Character Controller utilisent une entite Model separee avec un GameObject attache pour le rendu et l'animation. Le composant Animator se trouve sur ce GameObject, pas sur l'entite ECS.

    Architecture

    Entite Locomotor (ECS) → CharacterFollower → Entite Model (ECS) → GameObject (Animator)
    1. L'entite Locomotor execute la logique de la state machine dans des jobs Burst
    2. L'entite Model suit le Locomotor via CharacterFollowerSystem
    3. Le GameObject du Model a son transform synchronise via EntityGoTransformSyncSystem
    4. Un MonoBehaviour sur le GameObject lit l'etat de l'entite et pilote l'Animator

    Configuration

    1. Creer le GameObject du modele

    Dans la sous-scene :

    1. Creez un GameObject enfant avec votre mesh de personnage et un Animator
    2. Ajoutez un composant CharacterFollowerAuthoring
    3. Definissez le champ Locomotor pour referencer le composant d'authoring du Locomotor du personnage

    2. Creer un MonoBehaviour pont d'animation

    Etant donne que les jobs Burst ne peuvent pas acceder a l'Animator, vous avez besoin d'un MonoBehaviour pont qui lit l'etat ECS et definit les parametres de l'Animator :

    public class LocomotionAnimationBridge : MonoBehaviour
{
    Animator _animator;
    EntityGameObject _entityLink;

    // Hachages des parametres de l'Animator
    static readonly int SpeedHash = Animator.StringToHash("Speed");
    static readonly int IsGroundedHash = Animator.StringToHash("IsGrounded");
    static readonly int IsJumpingHash = Animator.StringToHash("IsJumping");

    void Awake()
    {
        _animator = GetComponent<Animator>();
        _entityLink = GetComponent<EntityGameObject>();
    }

    void LateUpdate()
    {
        if (_entityLink == null || !_entityLink.IsValid) return;

        // Lire les donnees de l'entite depuis le monde ECS
        var entityManager = World.DefaultGameObjectInjectionWorld.EntityManager;
        var entity = _entityLink.Entity;

        // Lire la velocite pour le parametre de vitesse
        if (entityManager.HasComponent<LocomotionVelocity>(entity))
        {
            var velocity = entityManager.GetComponentData<LocomotionVelocity>(entity);
            _animator.SetFloat(SpeedHash, math.length(velocity.Value));
        }

        // Lire le contact pour l'etat au sol
        if (entityManager.HasComponent<LocomotionContact>(entity))
        {
            var contact = entityManager.GetComponentData<LocomotionContact>(entity);
            _animator.SetBool(IsGroundedHash,
                contact.ContactType == ContactType.Ground);
        }
    }
}

    3. Attacher au GameObject du modele

    Ajoutez LocomotionAnimationBridge au meme GameObject qui possede l'Animator. Le composant EntityGameObject est ajoute automatiquement par la couche hybride lors de l'initialisation.

    Animation tenant compte de l'etat

    Pour lire l'etat actuel de la state machine, vous avez besoin d'acceder au composant genere de la state machine. Comme il s'agit d'un type generique, vous pouvez soit :

    Option A : Utiliser un managed component ou une structure de donnees partagee pour exposer l'etat actuel aux MonoBehaviours.

    Option B : Creer un systeme qui ecrit les donnees pertinentes pour l'animation dans un composant "donnees d'animation" dedie, lisible depuis le pont.

    L'option B est recommandee car elle minimise le travail sur le thread principal :

    public struct AnimationData : IComponentData
{
    public float Speed;
    public bool IsGrounded;
    public bool IsJumping;
    public bool IsFalling;
}

[UpdateAfter(typeof(MyLocomotionSystem))]
partial struct AnimationDataSystem : ISystem
{
    public void OnUpdate(ref SystemState state)
    {
        foreach (var (animData, velocity, contact) in
            SystemAPI.Query<RefRW<AnimationData>,
                RefRO<LocomotionVelocity>,
                RefRO<LocomotionContact>>())
        {
            animData.ValueRW.Speed = math.length(velocity.ValueRO.Value);
            animData.ValueRW.IsGrounded =
                contact.ValueRO.ContactType == ContactType.Ground;
        }
    }
}

    Limitations connues

    • L'animation est toujours en retard d'une frame par rapport a la physique car l'Animator s'execute apres que l'ECS ait synchronise le transform
    • Le root motion n'est pas supporte : c'est l'entite Locomotor qui pilote la position, pas l'Animator
    • Le blending entre les etats doit etre gere par l'Animator Controller, pas par la state machine de locomotion
