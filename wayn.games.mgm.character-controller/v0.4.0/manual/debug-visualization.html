    Debug Visualization

    The MGM Character Controller includes a built-in debug visualization system that renders contact points, movement paths, and collision impulses directly in the Scene view. This is invaluable for diagnosing movement issues, tuning physics parameters, and understanding what the locomotion system is doing each frame.

    Enabling Debug Visualization

    Debug visualization is opt-in per entity:

    1. Select the character entity's authoring GameObject in the sub-scene
    2. Add a LocomotionDebugConfigAuthoring component
    3. Configure which visualizations you want to see
    4. Enter Play mode

    The debug config is baked into a blob asset, so changes require re-entering Play mode.

    Debug Data Types

    Path Points

    Path points show the character's position at each stage of the execution loop. They visualize the trajectory the character takes through a single frame, including each bounce of the collide-and-slide algorithm.

    Contact Points

    Contact points mark where the character's collider touches the environment. They are color-coded by contact type:

    Contact Type What It Means
    Ground The surface is walkable (slope below MaxSlope)
    Wall The surface is too steep to walk on
    Ceiling The surface is above the character
    None No contact this frame

    Impulse Points

    Impulse points show where collision forces are applied, both forces the character receives and forces it applies to other bodies. These help diagnose unexpected velocity changes or interactions with dynamic objects.

    Debug Stages

    Each visualization point is tagged with the execution stage where it was recorded:

    Stage Description
    Start Before any processing
    SolveOverlaps After overlap resolution
    CollideAndSlide After collide-and-slide
    SnapToGround After the ground snapping step
    InheritDisplacement After platform velocity computation

    This lets you pinpoint exactly which stage is producing unexpected behavior.

    Components and Systems

    LocomotionDebugConfig

    A blob asset storing debug configuration per entity. Controls visualization through duration floats (PathSeconds, ContactSeconds, ImpulseSeconds) - a duration of zero or less disables that category - and per-stage/per-type color and shape settings.

    LocomotionDebugRecorder

    A static utility that records debug points into dynamic buffers during job execution. Called from within the locomotion job when debug buffers are present.

    Debug Buffers

    Three IBufferElementData types store the recorded points:

    • LocomotionDebugPathPoint - Position + stage + frame index
    • LocomotionDebugContactPoint - Position + normal + contact type + body type + frame index
    • LocomotionDebugImpulsePoint - Position + direction + magnitude + body type + is-applied flag + frame index

    LocomotionDebugDisplaySystem

    Reads the debug buffers and renders visualizations using the Unity.DebugDisplay API (Burst-compatible line and arrow drawing via PhysicsDebugDisplaySystem).

    LocomotionDebugFrameCounterSystem

    Maintains a global frame counter used to tag debug points. This allows the display system to distinguish between points from different frames and optionally show history.

    Performance Considerations

    Debug visualization adds overhead:

    • Memory: Each entity with debug enabled allocates three dynamic buffers that grow each frame
    • CPU: Recording points in the locomotion job adds branch checks and buffer writes
    • Rendering: Drawing gizmos happens on the main thread

    For this reason:

    • Only enable debug on entities you are actively investigating
    • Disable debug visualization in builds
    • The debug components are automatically excluded when the authoring component is not present

    Custom Debug Inspector

    The LocomotionDebugConfigAuthoringEditor provides a custom Inspector for the debug config authoring component. It shows a visual preview of the configured debug options including duration settings, colors, and contact shapes.

    Tips for Debugging

    Character falls through ground

    Enable path points and ground snap visualization. Check if:

    • The snap-to-ground capsule cast is reaching the ground (path point should show a downward segment)
    • The contact type at the end of the frame is Ground

    Character gets stuck on walls

    Enable contact points. Check if:

    • The contact is classified as Wall vs Ground
    • The MaxSlope threshold is correct
    • Step detection is attempting and failing (check MaxStepHeight)

    Character launched unexpectedly

    Enable impulse points. Check if:

    • External impulses are being applied (from CharacterContactSystem)
    • A dynamic body collision is injecting unexpected velocity
    • Platform inheritance is double-applied

    State transitions not firing

    This is a logic issue, not a physics issue. Add Debug.Log calls in your transition helper methods (temporarily, for non-Burst debugging) or check the state machine component in the Entity Inspector.

    Profiler Integration

    Beyond visual debugging, the package integrates with Unity's Profiler:

    ProfilerMarkerExtensions

    Burst-compiled profiler marker utilities that add custom markers to the Profiler timeline. These markers show up under the locomotion job's execution, helping you identify which stage is consuming the most time.

    LocomotionProfilerHUD

    An optional MonoBehaviour (in the Hybrid assembly) that renders a simple in-game profiler overlay showing FPS and locomotion system timing.
