    Recipe: Add Air Control

    Allow the character to steer horizontally while airborne (jumping or falling).

    When to Use

    You want the player to influence the character's horizontal direction while in the air, rather than committing to a fixed trajectory on launch.

    Prerequisites

    • A working Falling state
    • Horizontal movement logic in MovingState (to reuse)

    Steps

    1. Reuse MovingState's horizontal acceleration in FallingState

    The sample already does this. In your FallingState.OnIntegrateState, call MovingState.ComputeHorizontalMovementAcceleration:

    public void OnIntegrateState(ref MyContext ctx)
{
    // Gravity
    FallingState.ComputeGravityAcceleration(ref ctx,
        ctx.Profile.Falling.TerminalVelocity,
        out float3 gravityAccel,
        ctx.Profile.Falling.GravityMultiplier);

    // Air control - same horizontal logic as ground movement
    ref BaseLocomotionContext baseContext = ref ctx.BaseContext;
    MovingState.ComputeHorizontalMovementAcceleration(
        ref baseContext, ctx.Profile.Moving,
        ctx.Input.MovementDirection, ctx.Input.MovementMagnitude,
        out float3 moveAccel);

    baseContext.UpdateVelocitiesAndComputeDisplacements(moveAccel, gravityAccel);
}

    2. (Optional) Use a separate air control profile

    For reduced air control, create an AirControlProfile with lower acceleration:

    [Serializable]
public struct AirControlProfile
{
    public float MaxSpeed;
    public float TimeToMaxSpeed;
    public float TimeToStop;
    public float Sharpness;
}

    Then pass it instead of ctx.Profile.Moving:

    MovingState.ComputeHorizontalMovementAcceleration(
    ref baseContext, ctx.Profile.AirControl, // Slower acceleration
    ctx.Input.MovementDirection, ctx.Input.MovementMagnitude,
    out float3 moveAccel);

    3. Apply the same logic in JumpingState

    If you want air control during jumping too, add the same horizontal acceleration code to JumpingState.OnIntegrateState. The sample already does this.

    Expected Result

    The character can change direction while airborne. With a separate air control profile, the steering feels less responsive than ground movement.

    Common Mistakes

    • Using the full ground MovingProfile for air control - the character turns too sharply in the air
    • Forgetting to apply air control in both Jumping and Falling - the character can steer while falling but not while ascending
