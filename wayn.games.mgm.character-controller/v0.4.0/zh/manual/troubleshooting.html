    故障排除

    本页面按症状列出常见问题。对于每个问题：你看到什么、为什么发生、如何修复。

    角色穿过地板

    原因：地板没有 ECS 世界可以看到的物理碰撞体。

    修复：

    1. 在子场景内选择地板 GameObject
    2. 添加 Physics Shape 组件（不是旧版的 MeshCollider 或 BoxCollider）
    3. 验证子场景已加载（检查 Hierarchy 中的子场景图标）

    验证：进入 Play 模式。打开 Physics Debugger（Window > Analysis > Physics Debugger）并确认地板碰撞体可见。

    预防：始终对 ECS 碰撞体使用 Physics Shape / Physics Body 组件。旧版碰撞体会被 Unity.Physics 忽略。

    角色不响应输入

    原因：输入没有从硬件流向 ECS 组件。

    修复 - 按顺序检查每一层：

    1. Input System 包：确认 com.unity.inputsystem 已安装，且 Active Input Handling 在 Edit > Project Settings > Player 中设置为 Input System Package (New) 或 Both
    2. Player 实体：验证 Player 实体在 Player Character Authoring 组件中分配了输入资产
    3. Player 预制体：Player GO Prefab 必须有一个 PlayerInput 组件并分配了输入动作
    4. 角色引用：Player 实体必须引用正确的 Character 实体

    验证：在你的 PlayerInputAsset.ReadInputsFromAsset() 方法中添加断点或日志以确认它每帧被调用。

    角色在地面上抖动

    原因：角色的视觉位置每个固定步离散更新，导致可见的卡顿。

    修复：

    1. 在 Character Physics Profile 中将 InterpolationMode 设为 Interpolated
    2. 如果仍然抖动，略微增加 CollisionTolerance（尝试 0.01）
    3. 验证地面碰撞体是静态的（不是每帧重建的动态物体）

    验证：抖动应在 60+ FPS 下消失或变得不可察觉。

    角色穿过墙壁

    原因：碰撞滑动算法没有足够的迭代次数来解析碰撞。

    修复：

    1. 在 Character Physics Profile 上增加 MaxBounce（尝试 8-10，默认为 6）
    2. 检查墙壁的碰撞层是否包含在角色的 CollidesWith 掩码中
    3. 如果角色移动非常快，每帧的位移可能超过碰撞体尺寸——降低速度或提高固定更新率

    验证：启用调试可视化的接触点以查看墙壁接触是否被检测到。

    角色无法攀爬台阶

    原因：台阶检测条件未满足。

    修复：

    1. 在 Character Physics Profile 上增加 MaxStepHeight（必须小于胶囊体的平坦部分高度：CapsuleHeight - 2 * CapsuleRadius）
    2. 确保 MaxSlope 足够高使台阶的垂直面被分类为墙壁（而非地面）
    3. 验证台阶几何体是干净的——倾斜的网格伪影可能会混淆台阶检测器

    验证：启用调试可视化。墙壁接触应出现在台阶面，随后是成功的向上移动。

    角色从可行走的斜坡上滑落

    原因：坡度角超过了 MaxSlope 阈值。

    修复：在 Character Physics Profile 上增加 MaxSlope。默认在 45-60 度范围内处理大多数情况。

    验证：记录或检查 ctx.BaseContext.LocomotionContact.ValueRO.Normal 以计算实际坡度角。

    跳跃不触发

    原因：跳跃触发器输入被覆盖而不是累积，或者使用后未被消费。

    修复：

    1. 在你的 ICharacterLocomotionInput.ApplyInputsToCharacter 中，累积触发器：JumpTrigger += playerInputs.JumpTrigger（不是 JumpTrigger = playerInputs.JumpTrigger）
    2. 在你的 PlayerInputAsset.ReadInputsFromAsset 中，只在按钮被按下的帧返回 1f（WasPerformedThisFrame()），而不是在按住时

    验证：在 ApplyInputsToCharacter 中添加临时日志以查看累积值。

    跳跃每帧都触发

    原因：跳跃触发器使用后未被消费（重置为零）。

    修复：在状态的 OnIntegrateState 中读取触发值后重置它：

    LocomotionInput value = ctx.Input;
value.JumpTrigger = 0;
ctx.Input = value;

    验证：角色应该每次按下按钮跳一次，而不是连续跳。

    状态转换从不触发

    原因：转换的条件未满足，或者优先级更高的转换先触发了。

    修复：

    1. 检查转换上的 [EvaluateBefore] 属性——它们控制评估顺序。第一个匹配的转换胜出。
    2. 验证 ShouldTransition 方法中的条件是否匹配实际运行时状态。使用 Entity Inspector 检查 LocomotionContact、LocomotionVelocity 和你的输入组件值。
    3. 如果使用内联的 EvaluateTransitions 模式，检查 if 检查的顺序——较早的检查具有更高的优先级。

    验证：在转换的 ShouldTransition 内添加临时日志以查看哪个条件失败。

    角色意外弹射

    原因：动态物体碰撞施加了外部冲量，或者平台速度被应用了两次。

    修复：

    1. 启用冲量调试可视化以查看力的来源
    2. 检查你是否没有在 OnIntegrateState 中手动添加 PlatformHorizontalVelocity——系统在 ComputeInheritedDisplacement 步骤中自动处理平台继承
    3. 如果动态物体碰撞注入了过多速度，检查 Character Physics Profile 上的 Mass 和 MassToDynamic 值

    验证：通过在状态机结构体上将 HandleExternalImpulses 重写为空操作来临时禁用冲量处理。

    角色穿过或卡在移动平台上

    原因：地面吸附距离对于平台的速度来说太小。

    修复：

    1. 重写 context 上的 ComputeGroundSnappingDistance（或 OverrideGroundSnappingDistance partial 方法）以考虑平台的垂直速度
    2. 参见示例的 LocomotionContextExtensions.cs 了解添加 platformDistance 的参考实现

    验证：启用路径点调试可视化以查看角色是否在快速平台上失去地面接触。

    源代码生成器未生成代码

    原因：源代码生成器无法找到或处理你的状态机属性。

    修复：

    1. 确保你的状态枚举有 [LocomotionStateMachine] 属性
    2. 确保每个枚举值有 [LocomotionState(typeof(YourState))] 属性
    3. 检查你的项目是否引用了 Wayn.Games.Locomotion.Runtime 程序集
    4. 重启 Unity 或重新导入包——源代码生成器在编译时运行

    验证：检查你的程序集定义中的 Roslyn Analyzers 部分以确认源代码生成器 DLL 被引用。

    构建编译成功但角色在 Player 中不工作

    原因：源代码生成器使用的泛型类型可能未注册为 AOT 编译。

    修复：

    1. 确保你的程序集中有 [assembly: RegisterGenericComponentType(typeof(LocomotionInputAsset<YourInput>))]
    2. 检查构建日志中的 Burst 编译错误——状态方法必须是 Burst 兼容的（没有托管类型）
    3. 验证状态结构体或 context 中没有托管类型（string、class、数组）

    验证：启用 Development Build 进行构建并检查 Player 日志中的 Burst 或 ECS 错误。

    调试可视化不显示

    原因：调试组件未添加或 Scene 视图不可见。

    修复：

    1. 在子场景中选择角色实体的 authoring GameObject
    2. 添加 LocomotionDebugConfigAuthoring 组件
    3. 在 Inspector 中配置要启用的可视化选项
    4. 进入 Play 模式并切换到 Scene 视图（调试 Gizmo 不在 Game 视图中渲染）

    验证：调试数据被烘焙为 blob 资产，因此更改需要重新进入 Play 模式。
