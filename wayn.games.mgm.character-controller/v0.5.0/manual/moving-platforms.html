    Moving Platforms

    The MGM Character Controller has built-in support for moving and rotating platforms. Characters standing on platforms automatically inherit their velocity and rotation without any special logic in your state code.

    How It Works

    Moving platforms are kinematic physics bodies. The locomotion system detects when a character is grounded on any dynamic body (kinematic or fully-simulated) and automatically computes the displacement and rotation the character should inherit.

    Contact Detection

    During the SnapToGround step, the system detects that the character is standing on a dynamic body. It records the body index and entity reference in the LocomotionContact component.

    InheritDisplacementFromMotionVelocity

    At the end of the locomotion job, the InheritDisplacementFromMotionVelocity step reads the body's motion velocity from the physics world and decomposes it into:

    • PlatformHorizontalVelocity - The body's velocity projected onto the horizontal plane
    • PlatformVerticalVelocity - The body's velocity along the gravity axis

    The displacement is applied to the character's position in the same frame. The platform rotation is applied directly via a twist quaternion computation. The inherited velocities are available immediately in the same frame's EvaluateTransition step (e.g., a jumping state's OnEnterState can read PlatformHorizontalVelocity to preserve momentum when jumping off a platform).

    Important

    You should never add platform velocity manually in OnIntegrateState. The system handles it automatically. Adding it yourself would result in double-application, making the character move twice as fast on platforms.

    Platform Components

    PlatformMover

    Stores the movement speed and current waypoint index for track-based platform movement. Updated by the PlatformMovementSystem.

    Rotor

    Stores angular velocity for rotating platforms. The system applies the rotation each fixed update.

    Track

    Constrains platform motion along a predefined path. The platform loops cyclically through waypoints.

    For a step-by-step setup guide, see Recipe: Set Up a Moving Platform.

    PlatformMovementSystem

    The PlatformMovementSystem runs in the BeforePhysicsSystemGroup and computes the desired linear and angular velocities based on PlatformMover and Rotor components. It writes to the PhysicsVelocity component only - Unity.Physics then integrates those velocities to update the platform's transform during the physics step.

    Character Behavior on Platforms

    Standing Still

    When idle on a platform, the character moves with the platform. If the platform rotates, the character's rotation is updated directly by the InheritDisplacementFromMotionVelocity step via a twist quaternion computation (see the sample's IdleState.OnIntegrateState for the spin/roll/slide interpolation logic).

    Moving on a Platform

    When walking on a moving platform, the character's own velocity is additive with the platform's velocity. The character moves relative to the platform surface.

    Jumping off a Platform

    When the character jumps, the JumpingState.OnEnterState adds PlatformHorizontalVelocity to the character's horizontal velocity:

    ctx.BaseContext.HorizontalVelocity += ctx.BaseContext.PlatformHorizontalVelocity;

    This preserves the platform's momentum, so jumping off a fast-moving platform launches the character in the direction the platform was moving. Without this, the character would appear to "snap back" to a standstill mid-air.

    Platform Push (Overlap Resolution)

    If a platform moves into the character (e.g., a rising elevator ceiling), the SolveOverlaps step pushes the character out. If the character is trapped between two surfaces (e.g., elevator ceiling and floor), the overlap state transitions through Push -> Pinch -> Crush.

    When the character is pushed into geometry, the collide-and-slide algorithm can still move it back out because backface collisions are ignored. This means a character that has been shoved inside a wall by a platform can escape by moving away from the wall - the capsule cast will not collide with the interior face of the geometry.

    Custom Platform Behaviors

    You can create custom platform movement by writing your own system that modifies the PhysicsVelocity component on platform entities. The locomotion system will automatically handle character inheritance for any dynamic body the character is grounded on.

    Important

    Your system must run in BeforePhysicsSystemGroup so that the velocities are set before Unity.Physics integrates them into the platform's transform.

    For example, a platform that follows a spline:

    [BurstCompile]
[UpdateInGroup(typeof(BeforePhysicsSystemGroup))]
partial struct SplinePlatformSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        float dt = SystemAPI.Time.fixedDeltaTime;
        foreach (var (velocity, spline, transform) in
            SystemAPI.Query<RefRW<PhysicsVelocity>, RefRW<SplineFollower>, RefRO<LocalTransform>>()
)
        {
            float3 targetPos = EvaluateSpline(spline.ValueRO, SystemAPI.Time.ElapsedTime);
            velocity.ValueRW.Linear = (targetPos - transform.ValueRO.Position) / dt;
        }
    }
}
