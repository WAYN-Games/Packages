    扩展包

    MGM Character Controller 设计为无需修改包源码即可扩展。本页面介绍常见的扩展点和模式。

    扩展点

    1. 自定义状态

    最常见的扩展。通过创建新的状态结构体并将它们添加到状态枚举来添加新的移动行为（墙壁奔跑、游泳、攀爬、冲刺）。

    参见 编写自定义状态 获取完整指南。

    2. 自定义 Profile

    你的 ILocomotionProfile 结构体可以包含任何非托管数据。为新的游戏参数添加字段：

    [Serializable]
public partial struct MyProfile : ILocomotionProfile
{
    public MovingProfile Moving;
    public FallingProfile Falling;
    public JumpingProfile Jumping;

    // Custom extension
    public SwimmingProfile Swimming;
    public DashProfile Dash;
    public WallRunProfile WallRun;
}

    每个子 profile 都是一个带有配置字段的普通结构体。数据被烘焙到 blob 资产中，因此需要在你的 LocomotionProfileAsset 和 BuildBlobAsset 方法中添加对应字段。

    3. 自定义 Context

    你的 context 结构体可以包含额外的计算属性和辅助方法：

    public partial struct MyContext : ILocomotionContext<MyProfile, MyInput, MyStateData>
{
    public BaseLocomotionContext BaseContext;
    public MyStateData StateData;

    // Standard interface properties...

    // Custom helpers
    public bool IsUnderwater => BaseContext.CharacterPosition.Position.y < WaterLevel;
    public bool WantsToMove => Input.MovementMagnitude > math.EPSILON;
    public bool CanDash => StateData.DashCooldown <= 0;

    // Custom ground snap for swimming (no ground snap when submerged)
    public void ComputeGroundSnappingDistance(in MyProfile profile, out float distance)
    {
        if (IsUnderwater)
        {
            distance = 0; // No ground snapping while swimming
            return;
        }
        // Normal ground snap calculation...
    }
}

    4. 自定义 StateData

    添加每个状态的持久化数据，用于任何特定于状态的追踪：

    public partial struct MyStateData : IComponentData
{
    public JumpData JumpData;
    public float DashCooldown;
    public float WallRunTimer;
    public float3 WallNormal;  // Remembered from wall run entry
}

    状态数据在帧和状态转换之间持久存在。在 OnEnterState 或 OnExitState 中适当地清理它。

    5. 自定义输入

    为新动作添加输入通道：

    public partial struct MyInput : ICharacterLocomotionInput<MyInput>
{
    public float3 Movement;
    public float JumpTrigger;
    public float DashTrigger;      // New action
    public float InteractTrigger;  // New action

    public void ApplyInputsToCharacter(in MyInput playerInputs)
    {
        Movement = playerInputs.Movement;
        JumpTrigger += playerInputs.JumpTrigger;       // Accumulate triggers
        DashTrigger += playerInputs.DashTrigger;
        InteractTrigger += playerInputs.InteractTrigger;
    }
}

    记得在你的 PlayerInputAsset 子类中读取新输入，并在状态逻辑中消费触发器。

    6. 自定义地面吸附

    重写 context 上的 ComputeGroundSnappingDistance 以控制角色何时以及多远吸附到地面：

    public void ComputeGroundSnappingDistance(in MyProfile profile, out float distance)
{
    // Disable ground snapping while jumping upward
    if (StateData.CurrentState == MyStates.Jumping &&
        math.dot(BaseContext.VerticalVelocity, -BaseContext.GravityDirection) > 0)
    {
        distance = 0;
        return;
    }

    // Default calculation
    var slopeDistance = math.length(BaseContext.HorizontalVelocity * BaseContext.TimeStep) *
                       math.tan(math.radians(CharacterPhysicsProfile.MaxSlope));
    distance = math.max(CharacterPhysicsProfile.MaxStepHeight, slopeDistance);
}

    7. 自定义碰撞响应

    重写 OnResolveStateCollisions 以在角色碰到某物时添加自定义行为：

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();

    // Custom: bounce off trampolines
    if (baseContext.IsGroundingContact &&
        HasTrampolineTag(baseContext.LocomotionContact.ValueRO.Entity))
    {
        baseContext.VerticalVelocity = -baseContext.VerticalVelocity * 1.5f;
    }

    ctx.BaseContext = baseContext;
}

    8. 自定义系统

    你可以添加系统在管线的任何位置修改运动组件：

    [UpdateInGroup(typeof(FixedStepSimulationSystemGroup))]
[UpdateBefore(typeof(MyLocomotionSystem))]
partial struct GravityZoneSystem : ISystem
{
    public void OnUpdate(ref SystemState state)
    {
        foreach (var gravity in SystemAPI.Query<RefRW<LocomotionGravity>>()
            .WithAll<InGravityZone>())
        {
            gravity.ValueRW.Gravity = new float3(0, -4.9f, 0); // Half gravity
        }
    }
}

    自定义执行管线

    执行循环中的每个步骤都是 ILocomotionStateMachine 上的默认接口方法（DIM）。你可以在状态机的 partial 结构体上重写其中任何一个来替换或包装默认行为。

    源代码生成器为你的状态机生成一个 partial 结构体（例如 MyStateMachine）。因为它是 partial 的，你可以添加第二个 partial 声明来重写任何 DIM：

    public partial struct MyStateMachine
{
    public void Integrate(ref MyStateMachine stateMachine, ref MyContext context)
    {
        // Skip movement entirely when crushed
        if (context.BaseContext.OverlapState == OverlapState.Crush)
            return;

        // Otherwise run the default per-state integration
        stateMachine.IntegratePerState(ref context);
    }
}

    可重写的管线方法

    方法 默认行为 重写以...
    Execute 编排完整管线（下面所有步骤按顺序执行） 替换整个执行循环
    HandleExternalImpulses 从物体接触中累积速度冲量 自定义冲量过滤或缩放
    Integrate 调用 IntegratePerState（被压碎时跳过） 添加积分前/后逻辑
    SolveOverlaps 多平面反碰撞算法 自定义重叠解析
    CollideAndSlide 循环 ResolveCollisionPerState 最多 MaxBounce 次 自定义碰撞迭代
    SnapToGround 向下胶囊投射以保持地面接触 自定义地面附着
    InheritDisplacementFromMotionVelocity 从物理体计算平台速度继承 自定义平台骑乘行为
    EvaluateTransition 调用 EvaluateTransitionPerState 添加转换前/后逻辑
    Note

    IntegratePerState、ResolveCollisionPerState 和 EvaluateTransitionPerState 由源代码生成器生成（switch 分发到每个状态的方法）。它们不打算被直接重写。

    参见 执行循环 了解这些步骤之间的顺序和数据流。

    自定义运动系统

    源代码生成器创建一个继承自 ILocomotionSystem<...> 的 partial 接口 I{Prefix}LocomotionSystem。生成的系统结构体使用 DIM 生命周期方法实现此接口，你可以重写这些方法。

    要自定义，添加你自己的 partial 接口声明：

    public partial interface IMyLocomotionSystem
{
    void ILocomotionSystem<MyStateMachine, MyStates, MyContext, MyProfile, MyInput, MyStateData>
        .OnLocomotionSystemUpdate(
            ref LocomotionJobData jobData, ref EntityQuery query, ref SystemState state)
    {
        if (query.IsEmptyIgnoreFilter) return;
        jobData = jobData.Update(ref state);

        // Schedule single-threaded instead of parallel (useful for debugging)
        state.Dependency = ScheduleLocomotionJob(
            state.Dependency,
            state.WorldUnmanaged.Time.DeltaTime,
            jobData, query,
            jobData.GetPhysicsWorldSingleton());
    }
}

    可重写的系统生命周期方法

    方法 默认行为 重写以...
    OnLocomotionSystemCreate 构建实体查询并创建 LocomotionJobData 添加 RequireForUpdate、自定义查询过滤器、速率管理器设置
    OnLocomotionSystemUpdate 更新作业数据并调用 ScheduleParallelLocomotionJob 使用单线程调度、添加条件跳过、集成速率管理器
    OnLocomotionSystemDestroy 无操作 自定义资源清理
    Note

    默认的 DIM 实现使用 ref 参数而不是 SystemAPI，因为 Unity 的源代码生成器无法修补包的源代码生成器发出的代码中的调用站点（两者并行运行）。你自己的 partial 接口重写可以使用 SystemAPI，因为它们对 Unity 的生成器可见。

    DIM 自定义参考

    包中所有默认接口方法的完整参考：

    接口 方法 默认 重写以...
    ILocomotionState OnEnterState 无操作 在进入时初始化特定于状态的数据
    ILocomotionState OnIntegrateState 应用重力 + 计算位移 每个状态的自定义移动逻辑
    ILocomotionState OnExitState 无操作 在退出时清理特定于状态的数据
    ILocomotionState OnResolveStateCollisions baseContext.Move() 每个状态的自定义碰撞响应
    ILocomotionContext ComputeGroundSnappingDistance 返回 1f 基于速度或状态的吸附距离
    ILocomotionStateMachine Execute 完整管线 替换整个执行循环
    ILocomotionStateMachine HandleExternalImpulses 累积接触冲量 自定义冲量处理
    ILocomotionStateMachine Integrate 调用 IntegratePerState 积分前/后逻辑
    ILocomotionStateMachine SolveOverlaps 多平面反碰撞 自定义重叠解析
    ILocomotionStateMachine CollideAndSlide 弹跳循环最多 MaxBounce 次 自定义碰撞循环
    ILocomotionStateMachine SnapToGround 向下胶囊投射 自定义地面附着
    ILocomotionStateMachine InheritDisplacementFromMotionVelocity 平台速度继承 自定义平台骑乘
    ILocomotionStateMachine EvaluateTransition 调用 EvaluateTransitionPerState 转换前/后逻辑
    ILocomotionSystem OnLocomotionSystemCreate 构建查询 + 作业数据 自定义系统初始化
    ILocomotionSystem OnLocomotionSystemUpdate 并行调度 自定义调度策略
    ILocomotionSystem OnLocomotionSystemDestroy 无操作 自定义资源清理
    ICharacterLocomotionInput ApplyInputsToCharacter 无操作 在固定更新之间累积/合并输入

    多个状态机

    你可以在同一项目中拥有多个独立的状态机类型。例如：

    • GroundLocomotionStates - 行走、奔跑、跳跃、下落
    • VehicleStates - 驾驶、制动、加速、空中
    • SwimmingStates - 水面、水下、潜水

    每个都有自己的枚举、context、profile、输入、状态数据和生成的系统。它们完全独立。

    要在状态机之间切换角色，你需要添加/移除相关组件（状态机组件、输入、profile、状态数据）。这是一个涉及结构性实体更改的高级模式。

    使用源代码生成器

    Roslyn 源代码生成器读取锚点结构体和状态结构体上的属性并生成：

    1. 带分发逻辑的状态机组件
    2. 输入读取器系统
    3. 运动系统上的 ISystem 生命周期方法（OnCreate、OnUpdate、OnDestroy）

    生成器源码位于 SourceGenerator/SrcGenProj~/。要修改生成的代码模式：

    1. 编辑 StateMachineSourceGenerator.cs
    2. 从 SrcGenProj~ 目录使用 dotnet build 构建
    3. DLL 被复制到 SourceGenerator/ 并由 Unity 加载
    Warning

    修改源代码生成器会影响项目中的所有状态机。请彻底测试。

    创建复合角色

    对于需要 ECS 运动和复杂视觉层级结构的角色，使用 CompositeEntityAuthoring 模式：

    1. Locomotor 实体承载所有物理和状态机组件
    2. Model 实体通过 CharacterFollower 跟随 Locomotor
    3. Model 的 GameObject 可以有带 Animator、粒子系统、音频源的子对象

    CharacterFollowerAuthoring 在烘焙期间将模型链接到 locomotor。在运行时，CharacterFollowerSystem 同步变换。

    性能注意事项

    扩展到大量角色

    运动作业在 chunk 之间并行运行。要最大化吞吐量：

    • 保持所有角色具有相同的原型（相同的组件集）
    • 使用共享碰撞体（除非需要，否则避免 ForceUniqueColliderAuthoring）
    • 利用共享 blob 资产进行 Profile（所有引用相同 profile 的角色共享内存）
    • 只对你正在调查的角色启用调试可视化

    Burst 编译

    状态方法在 Burst 编译的作业内运行，因此它们必须是 Burst 兼容的。阻止 Burst 编译的常见错误：

    • 使用 string 或 List<T> 类型
    • 调用托管虚方法（DIM 生命周期方法由源代码生成器分发，而非通过虚分发）
    • 访问 GameObject 或 MonoBehaviour API
    • 使用 Debug.Log（使用 [Conditional("ENABLE_UNITY_COLLECTIONS_CHECKS")] 保护）

    性能分析

    使用 Unity 的 Profiler 识别瓶颈：

    • 运动作业在 Jobs 时间线中显示为 LocomotionJob
    • 每阶段的 profiler 标记是 ILocomotionStateMachine 上的静态字段，通过 ProfilerMarkerExtensions 由 [Conditional("ENABLE_LOCOMOTION_PROFILER")] 控制
    • LocomotionProfilerHUD 在游戏内显示帧级计时
