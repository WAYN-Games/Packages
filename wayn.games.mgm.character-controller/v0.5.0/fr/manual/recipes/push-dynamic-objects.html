    Recette : Pousser des objets dynamiques lors d'une collision

    Appliquez des impulsions de velocite aux corps rigides dynamiques lorsque le personnage entre en collision avec eux.

    Quand l'utiliser

    Vous souhaitez que le personnage pousse des caisses, des tonneaux ou d'autres objets physiques dynamiques en marchant dessus.

    Prerequis

    • Une state machine fonctionnelle
    • Des corps rigides dynamiques dans votre scene avec PhysicsBody (configure en Dynamic)

    Etapes

    1. Redefinir OnResolveStateCollisions

    Dans les etats ou vous souhaitez le comportement de poussee (typiquement MovingState), ajoutez la logique de poussee apres Move() :

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();

    // Pousser les corps dynamiques au contact
    if (baseContext.IsInContact &&
        baseContext.LocomotionContact.ValueRO.BodyType == BodyType.Dynamic)
    {
        float3 pushVelocity = math.normalizesafe(baseContext.HorizontalVelocity)
            * math.length(baseContext.HorizontalVelocity) * 0.5f;

        // Appliquer une impulsion au corps dynamique
        baseContext.ApplyImpulse(
            ref baseContext,
            baseContext.LocomotionContact.ValueRO.BodyIndex,
            baseContext.LocomotionContact.ValueRO.Entity,
            baseContext.PhysicsWorld.GetRigidBodyIndex(baseContext.CharacterEntity),
            baseContext.CharacterEntity,
            pushVelocity);
    }

    ctx.BaseContext = baseContext;
}

    2. Ajuster la force de poussee

    Ajustez le multiplicateur (0.5f ci-dessus) en fonction du ressenti de votre jeu. Des valeurs plus elevees poussent les objets plus loin. La Mass du personnage et MassToDynamic sur le profil physique affectent egalement le comportement d'impulsion par defaut de HandleExternalImpulses.

    Resultat attendu

    Lorsque le personnage marche contre un corps dynamique, celui-ci est pousse dans la direction du deplacement du personnage. La force de poussee est proportionnelle a la vitesse du personnage.

    Erreurs courantes

    • Utiliser le pattern copie-modification-reecriture mais oublier ctx.BaseContext = baseContext a la fin : l'impulsion est perdue
    • Appliquer l'impulsion dans OnIntegrateState au lieu de OnResolveStateCollisions : les contacts ne sont pas encore disponibles pendant l'integration
