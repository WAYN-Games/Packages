    Visualisation de debogage

    Le MGM Character Controller inclut un systeme de visualisation de debogage integre qui affiche les points de contact, les trajectoires de mouvement et les impulsions de collision directement dans la vue Scene. C'est un outil precieux pour diagnostiquer les problemes de mouvement, ajuster les parametres physiques et comprendre ce que fait le systeme de locomotion a chaque frame.

    Activer la visualisation de debogage

    La visualisation de debogage est activee par entite :

    1. Selectionnez le GameObject d'authoring de l'entite personnage dans la sous-scene
    2. Ajoutez un composant LocomotionDebugConfigAuthoring
    3. Configurez les visualisations que vous souhaitez voir
    4. Entrez en mode Play

    La configuration de debogage est cuite dans un blob asset, les modifications necessitent donc de relancer le mode Play.

    Types de donnees de debogage

    Points de trajectoire

    Les points de trajectoire montrent la position du personnage a chaque etape de la boucle d'execution. Ils visualisent la trajectoire que le personnage emprunte au cours d'une seule frame, y compris chaque rebond de l'algorithme collide-and-slide.

    Points de contact

    Les points de contact marquent l'endroit ou le collider du personnage touche l'environnement. Ils sont codes par couleur selon le type de contact :

    Type de contact Signification
    Ground La surface est praticable (pente inferieure a MaxSlope)
    Wall La surface est trop raide pour marcher dessus
    Ceiling La surface est au-dessus du personnage
    None Aucun contact cette frame

    Points d'impulsion

    Les points d'impulsion montrent ou les forces de collision sont appliquees, tant les forces que le personnage recoit que celles qu'il applique aux autres corps. Ils aident a diagnostiquer les changements de velocite inattendus ou les interactions avec des objets dynamiques.

    Etapes de debogage

    Chaque point de visualisation est associe a l'etape d'execution ou il a ete enregistre :

    Etape Description
    Start Avant tout traitement
    SolveOverlaps Apres la resolution des chevauchements
    CollideAndSlide Apres le collide-and-slide
    SnapToGround Apres l'etape d'accrochage au sol
    InheritDisplacement Apres le calcul de la velocite de la plateforme

    Cela vous permet de localiser exactement quelle etape produit un comportement inattendu.

    Composants et systemes

    LocomotionDebugConfig

    Un blob asset stockant la configuration de debogage par entite. Controle la visualisation via des durees flottantes (PathSeconds, ContactSeconds, ImpulseSeconds) - une duree de zero ou moins desactive cette categorie - ainsi que des parametres de couleur et de forme par etape/type.

    LocomotionDebugRecorder

    Un utilitaire statique qui enregistre les points de debogage dans des tampons dynamiques lors de l'execution des jobs. Appele depuis le job de locomotion lorsque des tampons de debogage sont presents.

    Tampons de debogage

    Trois types IBufferElementData stockent les points enregistres :

    • LocomotionDebugPathPoint - Position + etape + index de frame
    • LocomotionDebugContactPoint - Position + normale + type de contact + type de corps + index de frame
    • LocomotionDebugImpulsePoint - Position + direction + magnitude + type de corps + drapeau is-applied + index de frame

    LocomotionDebugDisplaySystem

    Lit les tampons de debogage et effectue le rendu des visualisations en utilisant l'API Unity.DebugDisplay (dessin de lignes et de fleches compatible Burst via PhysicsDebugDisplaySystem).

    LocomotionDebugFrameCounterSystem

    Maintient un compteur de frames global utilise pour etiqueter les points de debogage. Cela permet au systeme d'affichage de distinguer les points de differentes frames et d'afficher eventuellement un historique.

    Considerations de performance

    La visualisation de debogage ajoute une surcharge :

    • Memoire : Chaque entite avec le debogage active alloue trois tampons dynamiques qui grossissent a chaque frame
    • CPU : L'enregistrement des points dans le job de locomotion ajoute des verifications de branchement et des ecritures dans les tampons
    • Rendu : Le dessin des gizmos se fait sur le thread principal

    Pour cette raison :

    • N'activez le debogage que sur les entites que vous etudiez activement
    • Desactivez la visualisation de debogage dans les builds
    • Les composants de debogage sont automatiquement exclus lorsque le composant d'authoring n'est pas present

    Inspecteur de debogage personnalise

    Le LocomotionDebugConfigAuthoringEditor fournit un Inspector personnalise pour le composant d'authoring de configuration de debogage. Il affiche un apercu visuel des options de debogage configurees, y compris les parametres de duree, les couleurs et les formes de contact.

    Conseils de debogage

    Le personnage tombe a travers le sol

    Activez les points de trajectoire et la visualisation de l'accrochage au sol. Verifiez si :

    • Le capsule cast d'accrochage au sol atteint le sol (le point de trajectoire devrait montrer un segment descendant)
    • Le type de contact a la fin de la frame est Ground

    Le personnage reste bloque sur les murs

    Activez les points de contact. Verifiez si :

    • Le contact est classifie comme Wall vs Ground
    • Le seuil MaxSlope est correct
    • La detection de marche tente et echoue (verifiez MaxStepHeight)

    Le personnage est propulse de maniere inattendue

    Activez les points d'impulsion. Verifiez si :

    • Des impulsions externes sont appliquees (depuis CharacterContactSystem)
    • Une collision avec un corps dynamique injecte une velocite inattendue
    • L'heritage de plateforme est applique en double

    Les transitions d'etat ne se declenchent pas

    C'est un probleme de logique, pas de physique. Ajoutez des appels Debug.Log dans vos methodes d'aide aux transitions (temporairement, pour le debogage hors Burst) ou verifiez le composant de la machine a etats dans l'Entity Inspector.

    Integration avec le Profiler

    Au-dela du debogage visuel, le package s'integre avec le Profiler de Unity :

    ProfilerMarkerExtensions

    Des utilitaires de marqueurs de profilage compiles avec Burst qui ajoutent des marqueurs personnalises a la timeline du Profiler. Ces marqueurs apparaissent sous l'execution du job de locomotion, vous aidant a identifier quelle etape consomme le plus de temps.

    LocomotionProfilerHUD

    Un MonoBehaviour optionnel (dans l'assembly Hybrid) qui affiche un overlay de profilage simple en jeu montrant les FPS et le temps du systeme de locomotion.
