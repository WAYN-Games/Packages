    Plateformes mobiles

    Le MGM Character Controller inclut une prise en charge native des plateformes mobiles et rotatives. Les personnages se tenant sur des plateformes heritent automatiquement de leur velocite et de leur rotation sans necessiter de logique speciale dans votre code d'etat.

    Fonctionnement

    Les plateformes mobiles sont des corps physiques cinematiques. Le systeme de locomotion detecte lorsqu'un personnage est au sol sur un corps dynamique (cinematique ou entierement simule) et calcule automatiquement le deplacement et la rotation que le personnage doit heriter.

    Detection de contact

    Lors de l'etape SnapToGround, le systeme detecte que le personnage se tient sur un corps dynamique. Il enregistre l'index du corps et la reference d'entite dans le composant LocomotionContact.

    InheritDisplacementFromMotionVelocity

    A la fin du job de locomotion, l'etape InheritDisplacementFromMotionVelocity lit la velocite de mouvement du corps depuis le monde physique et la decompose en :

    • PlatformHorizontalVelocity - La velocite du corps projetee sur le plan horizontal
    • PlatformVerticalVelocity - La velocite du corps le long de l'axe de gravite

    Le deplacement est applique a la position du personnage dans la meme frame. La rotation de la plateforme est appliquee directement via un calcul de quaternion de torsion. Les velocites heritees sont disponibles immediatement dans l'etape EvaluateTransition de la meme frame (par exemple, le OnEnterState d'un etat de saut peut lire PlatformHorizontalVelocity pour conserver la quantite de mouvement lors du saut depuis une plateforme).

    Important

    Vous ne devriez jamais ajouter manuellement la velocite de la plateforme dans OnIntegrateState. Le systeme la gere automatiquement. L'ajouter vous-meme entrainerait une double application, faisant se deplacer le personnage deux fois plus vite sur les plateformes.

    Composants de plateforme

    PlatformMover

    Stocke la vitesse de deplacement et l'index du waypoint actuel pour le mouvement de plateforme base sur un parcours. Mis a jour par le PlatformMovementSystem.

    Rotor

    Stocke la velocite angulaire pour les plateformes rotatives. Le systeme applique la rotation a chaque mise a jour fixe.

    Track

    Contraint le mouvement de la plateforme le long d'un chemin predefini. La plateforme boucle de maniere cyclique a travers les waypoints.

    Pour un guide de configuration etape par etape, voir Recette : Configurer une plateforme mobile.

    PlatformMovementSystem

    Le PlatformMovementSystem s'execute dans le BeforePhysicsSystemGroup et calcule les velocites lineaires et angulaires souhaitees en fonction des composants PlatformMover et Rotor. Il ecrit uniquement dans le composant PhysicsVelocity - Unity.Physics integre ensuite ces velocites pour mettre a jour la transformation de la plateforme lors de l'etape physique.

    Comportement du personnage sur les plateformes

    Immobile

    Lorsqu'il est immobile sur une plateforme, le personnage se deplace avec la plateforme. Si la plateforme tourne, la rotation du personnage est mise a jour directement par l'etape InheritDisplacementFromMotionVelocity via un calcul de quaternion de torsion (voir la logique d'interpolation spin/roll/slide dans IdleState.OnIntegrateState de l'exemple).

    En mouvement sur une plateforme

    Lorsqu'il marche sur une plateforme mobile, la velocite propre du personnage s'additionne a celle de la plateforme. Le personnage se deplace relativement a la surface de la plateforme.

    Saut depuis une plateforme

    Lorsque le personnage saute, le JumpingState.OnEnterState ajoute PlatformHorizontalVelocity a la velocite horizontale du personnage :

    ctx.BaseContext.HorizontalVelocity += ctx.BaseContext.PlatformHorizontalVelocity;

    Cela conserve la quantite de mouvement de la plateforme, de sorte que sauter depuis une plateforme rapide propulse le personnage dans la direction ou la plateforme se deplacait. Sans cela, le personnage semblerait "revenir brusquement" a l'arret en plein vol.

    Poussee de plateforme (resolution de chevauchement)

    Si une plateforme se deplace dans le personnage (par exemple, le plafond d'un ascenseur qui monte), l'etape SolveOverlaps pousse le personnage vers l'exterieur. Si le personnage est coince entre deux surfaces (par exemple, le plafond et le sol d'un ascenseur), l'etat de chevauchement transite par Push -> Pinch -> Crush.

    Lorsque le personnage est pousse dans la geometrie, l'algorithme collide-and-slide peut toujours le sortir car les collisions avec les faces arriere sont ignorees. Cela signifie qu'un personnage qui a ete pousse a l'interieur d'un mur par une plateforme peut s'echapper en s'eloignant du mur - le capsule cast n'entrera pas en collision avec la face interieure de la geometrie.

    Comportements de plateforme personnalises

    Vous pouvez creer un mouvement de plateforme personnalise en ecrivant votre propre systeme qui modifie le composant PhysicsVelocity sur les entites de plateforme. Le systeme de locomotion gerera automatiquement l'heritage pour tout corps dynamique sur lequel le personnage est au sol.

    Important

    Votre systeme doit s'executer dans BeforePhysicsSystemGroup afin que les velocites soient definies avant que Unity.Physics ne les integre dans la transformation de la plateforme.

    Par exemple, une plateforme qui suit une spline :

    [BurstCompile]
[UpdateInGroup(typeof(BeforePhysicsSystemGroup))]
partial struct SplinePlatformSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        float dt = SystemAPI.Time.fixedDeltaTime;
        foreach (var (velocity, spline, transform) in
            SystemAPI.Query<RefRW<PhysicsVelocity>, RefRW<SplineFollower>, RefRO<LocalTransform>>()
)
        {
            float3 targetPos = EvaluateSpline(spline.ValueRO, SystemAPI.Time.ElapsedTime);
            velocity.ValueRW.Linear = (targetPos - transform.ValueRO.Position) / dt;
        }
    }
}
