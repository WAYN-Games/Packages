    MGM Character Controller

    MGM Character Controller est un package modulaire et performant de locomotion de personnage pour Unity 6, entièrement construit sur la pile DOTS (Entities, Unity.Physics, Burst, Jobs). Il propose une approche du mouvement pilotée par machine à états avec génération de code source complète, le rendant à la fois facile à étendre et extrêmement rapide à l'exécution.

    Quand utiliser ce package

    Utilisez MGM Character Controller lorsque vous avez besoin de :

    • Un contrôleur de personnage cinématique construit sur Unity ECS
    • Un mouvement piloté par machine à états avec des hooks de cycle de vie clairs par état
    • Une locomotion parallèle compilée par Burst pour de nombreux personnages
    • Une séparation nette entre la simulation physique (ECS) et la présentation (GameObjects)

    Ce package n'est pas destiné aux projets qui utilisent des contrôleurs de personnage basés sur MonoBehaviour, qui n'utilisent pas Unity DOTS, ou qui ont besoin d'un remplacement direct du composant CharacterController intégré à Unity.

    Fonctionnalités clés

    • Locomotion ECS pure avec des jobs compilés par Burst pour un débit maximal
    • Mouvement piloté par machine à états avec un cycle de vie clair par état
    • Code généré automatiquement via des analyseurs Roslyn, pour vous concentrer sur la logique de gameplay
    • Physique collide-and-slide avec détection automatique de marches, gestion des pentes et accrochage au sol
    • Support des plateformes mobiles avec héritage de vélocité et gestion de la rotation
    • Couche hybride GameObject pour une intégration transparente avec les animations, l'audio et les caméras
    • Visualisation de débogage intégrée pour les points de contact, les trajectoires de mouvement et les impulsions de collision
    • Outils de scaffolding éditeur qui génèrent toute la structure de la machine à états en un clic

    Limitations

    • Les personnages sont uniquement cinématiques - pas de ragdoll ou de mouvement piloté par forces par défaut
    • Les modifications de profil nécessitent un re-baking (quitter et ré-entrer en mode Play)
    • La visualisation de débogage fonctionne uniquement dans l'éditeur et ajoute une surcharge lorsqu'elle est activée
    • Pas de pont de machine à états d'animation intégré (vous connectez votre Animator sur le GameObject du modèle)
