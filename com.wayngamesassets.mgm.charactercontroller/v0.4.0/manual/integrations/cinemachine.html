    Integration: Cinemachine

    The MGM Character Controller works with Cinemachine for third-person (and other) camera setups. The Player prefab holds the camera rig, and the system automatically connects it to the controlled character.

    Setup

    1. Player prefab

    The Player GO Prefab should contain:

    • A Camera component
    • A CinemachineBrain on the same camera GameObject
    • One or more Cinemachine virtual cameras (e.g., CinemachineCamera for third-person follow)
    • A PlayerGoAuthoring MonoBehaviour with references to the camera and a camera target transform

    2. Virtual camera targeting

    In your PlayerInputAsset.InitInputAsset, connect the virtual cameras to the character's camera target:

    public override void InitInputAsset(PlayerGameObjectInstance instance)
{
    var playerGo = instance.Instance.Value;
    var playerGoAuthoring = playerGo.GetComponent<PlayerGoAuthoring>();
    _camera = playerGoAuthoring.Camera;
    _playerInput = playerGo.GetComponent<PlayerInput>();

    // Connect Cinemachine cameras to the character's follow target
    var cameraTarget = playerGoAuthoring.CameraTarget;
    var virtualCameras = playerGo.GetComponentsInChildren<CinemachineVirtualCameraBase>();
    foreach (var vc in virtualCameras)
    {
        vc.Follow = cameraTarget;
        vc.LookAt = cameraTarget;
    }
}

    3. Camera target tracking

    The CameraRigFollowControlledCharacterSystem updates the camera target position to match the controlled character's position each frame. This happens automatically when the Player entity has a ControlledCharacters buffer.

    4. Camera-relative input

    In ReadInputsFromAsset, convert 2D input to world-space relative to the camera:

    Transform cam = _camera.transform;
float3 worldMove = cam.forward * moveRaw.y + cam.right * moveRaw.x;
worldMove -= math.projectsafe(worldMove, math.up());
inputs.Movement = math.normalizesafe(worldMove);

    Local Multiplayer

    For split-screen setups:

    1. Each Player entity gets its own Player GO Prefab instance
    2. Each instance has its own Camera + CinemachineBrain
    3. CinemachineInputAxisController components should have their PlayerIndex set to match the PlayerInput component's player index

    The PlayerInputAsset is instantiated per player, so each player caches their own camera reference.

    Known Caveats

    • The camera target transform is a GameObject synced via CameraRigFollowControlledCharacterSystem, which uses a Burst-compiled IJobParallelForTransform with TransformAccessArray.
    • Camera lag during fast movement can occur if the sync runs after Cinemachine's update. Ensure the sync system runs before Cinemachine evaluates.
