    Recipe: Push Dynamic Objects on Collision

    Apply velocity impulses to dynamic rigid bodies when the character collides with them.

    When to Use

    You want the character to push crates, barrels, or other dynamic physics objects when walking into them.

    Prerequisites

    • A working state machine
    • Dynamic rigid bodies in your scene with PhysicsBody (set to Dynamic)

    Steps

    1. Override OnResolveStateCollisions

    In the states where you want push behavior (typically MovingState), add push logic after Move():

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();

    // Push dynamic bodies on contact
    if (baseContext.IsInContact &&
        baseContext.LocomotionContact.ValueRO.BodyType == BodyType.Dynamic)
    {
        float3 pushVelocity = math.normalizesafe(baseContext.HorizontalVelocity)
            * math.length(baseContext.HorizontalVelocity) * 0.5f;

        // Apply impulse to the dynamic body
        baseContext.ApplyImpulse(
            ref baseContext,
            baseContext.LocomotionContact.ValueRO.BodyIndex,
            baseContext.LocomotionContact.ValueRO.Entity,
            baseContext.PhysicsWorld.GetRigidBodyIndex(baseContext.CharacterEntity),
            baseContext.CharacterEntity,
            pushVelocity);
    }

    ctx.BaseContext = baseContext;
}

    2. Tune the push force

    Adjust the multiplier (0.5f above) based on your game's feel. Higher values push objects further. The character's Mass and MassToDynamic on the physics profile also affect the default impulse behavior from HandleExternalImpulses.

    Expected Result

    When the character walks into a dynamic body, the body is pushed in the direction the character is moving. The push force scales with the character's speed.

    Common Mistakes

    • Using the copy-modify-writeback pattern but forgetting ctx.BaseContext = baseContext at the end - the impulse is lost
    • Applying impulse in OnIntegrateState instead of OnResolveStateCollisions - contacts are not available yet during integration
