    编写自定义状态

    状态是角色行为的构建块。每个状态定义角色如何移动、对物理做出反应以及转换到其他状态。本指南涵盖状态接口、生命周期方法和常见模式。

    状态接口

    每个状态都是一个实现 ILocomotionState 的非托管结构体：

    [LocomotionState(typeof(MyCharacterStateMachine))]
internal partial struct MyState
{
    public void OnEnterState(ref MyContext ctx) { }

    public void OnIntegrateState(ref MyContext ctx) { }

    public void OnResolveStateCollisions(ref MyContext ctx) { }

    public void OnExitState(ref MyContext ctx) { }
}

    关键规则：

    • 所有生命周期方法都是带有默认接口实现（DIM 模式）的实例方法。它们是 public void，不是 internal static。ILocomotionState 接口由源代码生成器通过 partial 机制添加。
    • 转换由单独的 ILocomotionTransition 结构体处理（见下方转换）——状态结构体不定义转换逻辑
    • 源代码生成器将调用分派到正确的状态结构体——运行时没有虚拟分派
    • 状态方法在已经 Burst 编译的 Job 内运行，因此状态结构体和方法上不需要显式 [BurstCompile]
    • 不能使用托管类型（字符串、类、数组）——此代码在 Burst Jobs 内运行

    生命周期方法

    OnEnterState

    当状态机转换进入此状态时调用一次。用于：

    • 初始化状态特定数据（重置计时器、计数器）
    • 设置初始速度
    • 清除接触信息（用于空中状态）
    public void OnEnterState(ref MyContext ctx)
{
    // 进入地面状态：取消所有垂直动量
    ctx.BaseContext.VerticalVelocity = float3.zero;
    ctx.BaseContext.VerticalDisplacement = float3.zero;
}

    OnIntegrateState

    在积分阶段每帧调用一次。这是您根据输入、重力和配置文件参数计算角色期望位移的地方。

    您的任务是在 BaseLocomotionContext 上填充 HorizontalDisplacement、VerticalDisplacement、HorizontalVelocity 和 VerticalVelocity。

    最简单的方式是计算加速度并调用 UpdateVelocitiesAndComputeDisplacements：

    public void OnIntegrateState(ref MyContext ctx)
{
    ref BaseLocomotionContext baseContext = ref ctx.BaseContext;

    // 计算重力
    float3 gravityAcceleration = baseContext.GravityDirection * baseContext.GravityStrength;

    // 从输入计算移动加速度
    float3 desiredVelocity = ctx.Input.MovementDirection * ctx.Profile.MaxSpeed;
    float accel = ctx.Profile.MaxSpeed / ctx.Profile.TimeToMaxSpeed;
    MathematicsHelpers.ComputeAccelerationToTargetVelocity(
        out float3 moveAcceleration,
        baseContext.HorizontalVelocity,
        desiredVelocity,
        baseContext.TimeStep,
        accel);

    // 应用两个加速度
    baseContext.UpdateVelocitiesAndComputeDisplacements(
        newHorizontalAcceleration: moveAcceleration,
        newVerticalAcceleration: gravityAcceleration);
}
    Important

    不要在此处添加平台速度。ComputeInheritedDisplacement 步骤在碰撞解析后自动处理。手动添加会导致双重应用。

    OnResolveStateCollisions

    每帧可能被调用多次（最多 MaxBounce 次迭代）。每次调用代表碰撞滑移算法的一次迭代。

    大多数情况下，您的实现只需委托给 baseContext.Move()：

    public void OnResolveStateCollisions(ref MyContext ctx)
{
    BaseLocomotionContext baseContext = ctx.BaseContext;
    baseContext.Move();
    ctx.BaseContext = baseContext;
}

    OnExitState

    当状态机转换离开此状态时调用一次。用于清理状态特定数据：

    public void OnExitState(ref MyContext ctx)
{
    // 离开跳跃状态时重置跳跃计数器
    ctx.StateData.JumpData.JumpsCount = 0;
}

    转换

    转换定义角色何时以及如何在状态之间移动。

    推荐：基于属性的转换

    将转换声明为带有 [LocomotionTransition] 的小型结构体。源代码生成器发现它们并自动处理分派、OnExitState 和 OnEnterState。

    [LocomotionTransition]
[EvaluateBefore(typeof(IdleToMoving))]
[EvaluateBefore(typeof(IdleToFalling))]
public struct IdleToJumping : ILocomotionTransition<IdleState, JumpingState>
{
    public bool ShouldTransition(ref MyContext ctx)
        => ctx.BaseContext.IsGroundingContact && ctx.Input.JumpTrigger > 0;
}

    关键规则：

    1. 结构体名称遵循 {Source}To{Target} 约定
    2. ILocomotionTransition<TFrom, TTo> 声明连接哪些状态
    3. ShouldTransition 在转换应该触发时返回 true
    4. [EvaluateBefore] / [EvaluateAfter] 控制优先级——第一个匹配的转换获胜
    5. 源代码生成器自动调用源状态的 OnExitState 和目标状态的 OnEnterState

    常见状态模式

    地面状态（Idle / Moving）

    • OnEnterState：将垂直速度归零
    • OnIntegrateState：仅计算水平加速度
    • 转换：检查跳跃输入、失去地面接触、坡度超过最大值

    空中状态（Falling / Jumping）

    • OnEnterState：清除接触（LocomotionContact.None），可选择继承平台速度
    • OnIntegrateState：应用重力 + 可选空中控制
    • 转换：检查地面接触、墙壁接触

    滑行状态

    • OnIntegrateState：应用重力（角色沿斜坡滑下）
    • 转换：检查失去接触（转为下落）、地面接触低于最大坡度（转为移动/空闲）
