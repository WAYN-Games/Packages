    Getting Started

    This guide walks you through installing the MGM Character Controller package and getting a playable character running in your scene.

    Prerequisites

    Before installing, make sure your project meets these requirements:

    • Unity 6000.3 or later
    • Entities 1.4.2 (with Burst and Jobs)
    • Unity.Physics 1.4.2
    • Unity.Transforms (included with Entities)

    The package is designed for Unity's Data-Oriented Technology Stack (DOTS). If you are new to ECS, review Unity's Entities documentation first. You do not need to be an ECS expert, but you should understand the basic concepts of entities, components, systems, and baking.

    Installation

    From a local folder

    1. Open your Unity project
    2. Go to Window > Package Manager
    3. Click the + button and select Add package from disk...
    4. Navigate to the package.json file inside the MGM Locomotion folder
    5. Click Open

    From a git URL

    1. Open Window > Package Manager
    2. Click + > Add package from git URL...
    3. Enter the repository URL pointing to the Assets/MGM Locomotion folder

    Project Setup

    After installation, your project needs a sub-scene with physics enabled. The package relies on Unity.Physics for all collision detection and does not use the legacy physics system.

    1. Create or open a scene
    2. Add a Sub Scene (right-click in Hierarchy > New Sub Scene)
    3. Ensure the sub-scene has a Physics Step component (automatically present in most setups)
    4. Add static colliders to your environment (floor, walls, etc.)

    Creating Your First Character

    The fastest way to get started is to use the built-in scaffolding tool:

    Step 1: Generate the State Machine

    1. In the Project window, right-click on the folder where you want to create your state machine
    2. Select Assets > Create > WAYN Games > Locomotion > StateMachine
    3. Name the folder (e.g., MyCharacter)
    4. Press Enter

    The tool generates the entire folder structure with all required scripts:

    MyCharacter/
├── Runtime/
│   ├── MyCharacter.runtime.asmdef              # Auto-generated assembly definition
│   ├── States/
│   │   ├── MyCharacterStates.cs                # State enum with source generator attributes
│   │   └── MyCharacterState.cs                 # Initial state implementation
│   ├── Data/
│   │   ├── MyCharacterContext.cs                # Locomotion context struct
│   │   ├── MyCharacterProfile.cs                # Profile data struct
│   │   ├── MyCharacterInput.cs                  # Input component struct
│   │   ├── MyCharacterStateData.cs              # Per-state persistent data
│   │   ├── MyCharacterProfileAsset.cs           # Profile ScriptableObject
│   │   └── MyCharacterPlayerInputAsset.cs       # Input ScriptableObject
│   └── Systems/
│       ├── IMyCharacterLocomotionSystem.cs      # Locomotion system interface
│       └── MyCharacterLocomotionSystem.cs       # System implementation
├── Authoring/
│   ├── MyCharacter.authoring.asmdef             # Auto-generated assembly definition
│   ├── MyCharacterCharacterLocomotorAuthoring.cs  # Character authoring
│   └── MyCharacterPlayerCharacterAuthoring.cs     # Player authoring
└── Editor/
    ├── MyCharacter.editor.asmdef                # Auto-generated assembly definition
    └── MyCharacterStateCreationMenu.cs          # Menu for adding new states

    Step 2: Create the Profile Asset

    1. In the Project window, right-click
    2. Select the Create menu item for your character profile asset (the menu path is generated for you)
    3. Configure movement speed, jump force, gravity multiplier, and other parameters in the Inspector

    Step 3: Create the Input Asset

    1. Right-click in the Project window
    2. Select the Create menu item for your player input asset
    3. Assign your Input Action References for movement and jump

    Step 4: Set Up the Character Prefab

    1. Create a new GameObject in your sub-scene
    2. Add the generated Character Locomotor Authoring component
    3. Assign your Profile Asset
    4. Configure the Physics Profile (capsule height, radius, mass, max slope, etc.)

    Step 5: Set Up the Player Prefab

    The Player is a separate concept from the Character. Players own input and control one or more characters.

    1. Create a GameObject outside the sub-scene (it is a regular GameObject, not an entity)
    2. Add a Camera, Cinemachine Brain, and Player Input component
    3. Add the generated Player Character Authoring component
    4. Assign the Character entity, the Input Asset, and the Player GO Prefab

    Step 6: Enter Play Mode

    Press Play. Your character should respond to input and interact with the physics environment. If the character falls through the floor, verify that:

    • Your floor has a physics collider
    • The character's collision layers (BelongsTo and CollidesWith) are configured correctly
    • The sub-scene is loaded

    Importing the Sample

    The package includes a SinglePlayerSample that demonstrates a complete setup with five states (Idle, Moving, Jumping, Falling, Sliding), a camera rig, moving platforms, and debug visualization.

    To study the sample:

    1. Navigate to Assets/MGM Locomotion/Samples/SinglePlayerSample/
    2. Open the scene in Scenes/
    3. Review the scripts in Scripts/SinglePlayerStateMachine/ for a complete reference implementation
    Tip

    The sample is the best resource for understanding how all the pieces connect. Read it alongside this documentation.
