    Physics Configuration

    The CharacterPhysicsProfile defines the physical properties of the character's collision capsule and how it interacts with the world. This page explains each parameter, its effect, and how to tune it.

    CharacterPhysicsProfile Fields

    Capsule Geometry

    Field Type Description
    CapsuleHeight float Total height of the collision capsule from bottom to top, in world units. The capsule's bottom sphere sits at y = CapsuleRadius and the top sphere at y = CapsuleHeight - CapsuleRadius.
    CapsuleRadius float Radius of both hemisphere caps. Also determines the minimum width of the character. Default: 0.5.

    The capsule geometry directly affects:

    • Which gaps the character can fit through
    • How close the character can get to walls
    • Step detection (contact height relative to capsule bottom)

    Mass and Forces

    Field Type Description
    Mass float Weight of the character. Used for collision response calculations when the character collides with dynamic rigid bodies. Higher mass means the character pushes objects more.
    MassToDynamic float Multiplier for the passive gravitational force the character exerts on dynamic objects. Controls how much the character's presence "pushes" nearby dynamic bodies.

    Movement Limits

    Field Type Description
    MaxSlope float Maximum slope angle (in degrees) the character can walk up. Surfaces steeper than this are treated as walls. Typical values: 45-60 degrees.
    MaxStepHeight float Maximum height (in world units) of obstacles the character can automatically step over. Must be less than CapsuleHeight - 2 * CapsuleRadius (the flat section of the capsule). Typical values: 0.3-0.5.
    MaxBounce int Maximum number of collide-and-slide iterations per frame. Each bounce allows the character to redirect along a surface it hits. Higher values handle more complex geometry (corners, narrow corridors) but cost more. Default: 6.

    Collision Layers

    Field Type Description
    BelongsTo LayerMask The physics layer(s) this character belongs to. Other colliders check this to decide if they can collide with the character.
    CollidesWith LayerMask The physics layer(s) this character tests against. The character only detects collisions with colliders on these layers.

    To make characters ignore each other, put them on a layer that does not appear in other characters' CollidesWith mask. To make characters collide with each other, include the character layer in both masks.

    Wall Interaction

    Field Type Description
    SpinOrRollAlongWall float [-1, 1] Controls character behavior on a rotating platform when pressed against a wall. At -1, the character spins with the platform while sliding. At 0, the character slides without rotating. At 1, the character "rolls" along the wall surface.

    Collision Tolerance

    Field Type Description
    CollisionTolerance float Skin thickness for all collision queries. This is the distance at which the system considers two surfaces "in contact." Decrease for tighter collision response; increase if you observe jitter or tunneling. Defaults to CollisionWorld.DefaultCollisionTolerance.

    Interpolation

    Field Type Description
    InterpolationMode enum Interpolated enables Unity.Physics graphical smoothing between fixed updates, producing smoother visual movement. NotInterpolated updates the transform directly each fixed step, which can appear jittery at low fixed rates but is simpler to debug.

    Default Values

    The static CharacterPhysicsProfile.Default property provides sensible starting values:

    CharacterPhysicsProfile.Default = new CharacterPhysicsProfile
{
    MaxBounce = 6,
    SpinOrRollAlongWall = 0,
    CapsuleRadius = 0.5f,
    CapsuleHeight = 2f,
    BelongsTo = 1,                                    // Layer 0 (Default)
    CollidesWith = ~0,                                 // All layers
    CollisionTolerance = CollisionWorld.DefaultCollisionTolerance
};

    Collision Response

    The character's capsule collider is created with CollisionResponsePolicy.CollideRaiseCollisionEvents. This means:

    1. The collider participates in collision detection
    2. Collision events are raised (used by CharacterContactSystem for impact detection)
    3. The character is not moved by Unity.Physics solver, the locomotion job handles all movement

    Shared vs Unique Colliders

    By default, capsule colliders are shared across entities with identical geometry and filter settings. This saves memory when you have many characters with the same shape.

    If you need a unique collider per entity (e.g., for runtime shape modification), add a ForceUniqueColliderAuthoring component to the character's authoring GameObject.

    Tuning Guide

    Character walks through walls

    • Increase MaxBounce (try 8-10)
    • Check CollidesWith includes the wall's layer
    • Increase CollisionTolerance slightly

    Character jitters on the ground

    • Enable InterpolationMode.Interpolated
    • Check that CollisionTolerance is not too small
    • Verify the ground collider is static (not rebuilding each frame)

    Character cannot climb steps

    • Increase MaxStepHeight
    • Ensure the step geometry is clean (not sloped mesh artifacts)
    • Check that MaxSlope is high enough that the step face is classified as a wall

    Character slides off slopes it should walk on

    • Increase MaxSlope (the threshold between "ground" and "wall")

    Character clips through fast-moving platforms

    • The ground snapping distance may be too small. Override ComputeGroundSnappingDistance in your context to account for higher platform speeds

    How the Collider Is Built

    During baking, the BaseCharacterLocomotorBaker constructs the capsule collider from your profile:

    1. Creates CapsuleGeometry from CapsuleHeight and CapsuleRadius
    2. Creates CollisionFilter from BelongsTo and CollidesWith
    3. Sets CollisionResponsePolicy.CollideRaiseCollisionEvents
    4. Hashes the geometry + filter to check if a matching blob already exists
    5. If not, creates a new CapsuleCollider blob asset
    6. Attaches it as a PhysicsCollider component

    LocomotionGravity

    Gravity is stored per-entity in the LocomotionGravity component, initialized to PhysicsStep.Default.Gravity during baking. The BaseLocomotionContext decomposes this into:

    • GravityDirection - Normalized direction vector (usually (0, -1, 0))
    • GravityStrength - Magnitude (usually 9.81)

    Your state logic reads these to apply gravity acceleration. You can modify the LocomotionGravity component at runtime to change gravity per-character (e.g., for gravity wells, low-gravity zones, or space sections).
