    Recette : Ajouter le double saut

    Permettez au personnage de sauter plusieurs fois en l'air en utilisant les données d'état pour suivre le nombre de sauts.

    Quand l'utiliser

    Vous souhaitez que le personnage puisse effectuer des sauts supplémentaires lorsqu'il est en l'air, jusqu'à un maximum configurable.

    Prérequis

    • Une machine à états fonctionnelle avec les états Jumping et Falling
    • L'entrée de saut déjà configurée

    Étapes

    1. Ajouter le suivi des sauts dans StateData

    public partial struct MyStateData : IComponentData
{
    public int JumpsCount;
}

    2. Ajouter le nombre maximum de sauts à votre profil

    [Serializable]
public struct JumpingProfile
{
    public float Force;
    public int MaxJumps; // Set to 2 for double jump, 3 for triple, etc.
}

    3. Suivre les sauts dans JumpingState

    [LocomotionState(typeof(MyStateMachine))]
internal partial struct JumpingState
{
    public void OnEnterState(ref MyContext ctx)
    {
        // Clear contact (we're airborne)
        ctx.BaseContext.LocomotionContact.ValueRW = LocomotionContact.None;
        // Inherit platform velocity
        ctx.BaseContext.HorizontalVelocity += ctx.BaseContext.PlatformHorizontalVelocity;
    }

    public void OnIntegrateState(ref MyContext ctx)
    {
        // Check for additional jumps
        if (ctx.Input.JumpTrigger > 0 &&
            ctx.StateData.JumpsCount < ctx.Profile.Jumping.MaxJumps)
        {
            // Apply jump force (resets vertical velocity for consistent jump height)
            ctx.BaseContext.VerticalVelocity =
                ctx.Profile.Jumping.Force * -ctx.BaseContext.GravityDirection;
            ctx.StateData.JumpsCount++;
        }

        // Consume the trigger
        LocomotionInput value = ctx.Input;
        value.JumpTrigger = 0;
        ctx.Input = value;

        // Apply gravity and air control
        FallingState.ComputeGravityAcceleration(ref ctx,
            ctx.Profile.Falling.TerminalVelocity,
            out float3 gravityAccel,
            ctx.Profile.Falling.GravityMultiplier);

        ref BaseLocomotionContext baseContext = ref ctx.BaseContext;
        baseContext.UpdateVelocitiesAndComputeDisplacements(float3.zero, gravityAccel);
    }

    public void OnExitState(ref MyContext ctx)
    {
        ctx.StateData.JumpsCount = 0; // Reset for next time
    }
}

    4. Incrémenter au premier saut (dans la transition)

    Le premier saut se produit lors de la transition vers JumpingState. L'état source de votre transition doit configurer le saut initial. Le compteur commence à 0 et le premier saut dans OnEnterState ou la transition le met à 1.

    Si vous utilisez le pattern [LocomotionTransition], la force de saut initiale est appliquée dans OnEnterState ou à la première frame de OnIntegrateState.

    5. Définir MaxJumps dans l'asset de profil

    Ouvrez votre ScriptableObject de profil et définissez Jumping > MaxJumps à 2 (ou plus).

    Résultat attendu

    Le personnage saute depuis le sol (saut 1) et peut appuyer à nouveau sur le saut en l'air (saut 2). Le compteur de sauts se réinitialise lorsque le personnage atterrit et quitte l'état de saut.

    Erreurs courantes

    • Ne pas réinitialiser JumpsCount dans OnExitState - le compteur se reporte à la séquence de sauts suivante
    • Ne pas consommer le déclencheur de saut - le personnage utilise tous les sauts en une seule frame
    • Définir MaxJumps = 1 et s'attendre à un double saut - MaxJumps est le nombre total incluant le premier saut, donc définissez-le à 2
