    快速入门

    本指南使用包含的示例，以最快的速度让您获得一个可运行的角色。

    目标：一个可以行走、跳跃和乘坐移动平台的胶囊角色。 时间：不到 5 分钟。

    前提条件

    • 已安装 MGM Character Controller（安装）
    • 一个带有 DOTS 包的空项目或现有 Unity 项目

    步骤 1：导入示例

    1. 打开 Window > Package Manager
    2. 在包列表中选择 MGM Character Controller
    3. 展开 Samples 部分
    4. 点击 SinglePlayerSample 旁边的 Import

    示例将被导入到 Assets/Samples/MGM Character Controller/0.4.0/SinglePlayerSample/。

    步骤 2：创建场景

    示例提供了脚本、预制件和资源，但没有场景。您需要创建一个：

    1. 创建新场景（File > New Scene > Basic (Built-in)）
    2. 将其保存在项目中的任意位置

    步骤 3：设置 Sub-Scene

    1. 在 Hierarchy 中右键点击并选择 New Sub Scene > Empty Scene
    2. 在 sub-scene 内创建地板：3D Object > Plane
    3. 将平面缩放为 (10, 1, 10) 以提供移动空间
    4. 为平面添加 Physics Shape 组件（这会为 ECS 世界创建物理碰撞体）

    步骤 4：添加角色

    1. 在 sub-scene 内创建一个名为 Character 的空 GameObject
    2. 添加 Locomotion Character Locomotor Authoring 组件（在 Add Component 中搜索 "LocomotionCharacterLocomotor"）
    3. 分配导入示例中的 SinglePlayerProfileAsset（Assets/Samples/.../SinglePlayerSample/Assets/SinglePlayerProfileAsset）
    4. 物理配置文件的默认值暂时可用（2 米高胶囊，0.5 米半径）

    步骤 5：添加玩家

    1. 在 sub-scene 内创建另一个名为 Player 的空 GameObject
    2. 添加 Locomotion Player Character Authoring 组件（在 Add Component 中搜索 "LocomotionPlayerCharacter"）
    3. 将 Character 字段设置为引用步骤 4 中的 Character GameObject
    4. 分配导入示例中的 SinglePlayerInputAsset
    5. 分配导入示例中的 Player (Prefab) 作为 Player GO Prefab

    步骤 6：按下 Play

    按下 Play。您应该看到：

    • 一个站在平面上的胶囊角色
    • WASD / 左摇杆移动角色
    • 空格键 / 手柄南键使角色跳跃
    • 摄像机跟随角色

    如果出了问题

    问题 修复方法
    角色穿过地板 确保平面有 Physics Shape 组件，而不仅是 MeshCollider
    角色不移动 检查 SinglePlayerInputAsset 是否已分配给 Player，以及 Unity 的 Input System 包是否已安装
    没有摄像机 检查 Player (Prefab) 是否已分配给 Player 实体
    Console 中关于缺少组件的错误 确保使用 Unity 6000.3+ 并安装了所有 DOTS 包

    发生了什么？

    烘焙期间创建了三个实体：

    1. Locomotor - 在纯 ECS 中运行的角色实体。Burst 编译的 Job 在每次固定更新中通过 7 步管线处理它：重叠解决、脉冲处理、积分（您的状态逻辑）、碰撞滑移、地面吸附、平台继承和转换评估。
    2. Player - 拥有输入资源和摄像机。PlayerGOManagerSystem 在运行时实例化了玩家预制件（摄像机 + 输入）。
    3. Model - 通过 CharacterFollower 跟随 Locomotor，并将其变换同步到 GameObject 进行渲染。

    示例定义了五个状态——Idle、Moving、Jumping、Falling 和 Sliding——涵盖了基本的角色运动。状态之间的转换声明为带有 [LocomotionTransition] 属性的小型结构体。
