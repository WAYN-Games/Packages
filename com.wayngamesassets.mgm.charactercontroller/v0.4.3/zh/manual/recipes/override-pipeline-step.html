    教程：重写管线步骤

    通过在状态机结构体上重写 DIM 来替换或包装执行循环中的某个步骤。

    适用场景

    您需要在管线级别实现自定义行为——例如，当角色被挤压时跳过积分步骤，在某个步骤前后添加逻辑，或完全替换地面吸附。

    前提条件

    • 一个正常工作的状态机
    • 熟悉执行循环和扩展包

    步骤

    1. 为状态机添加部分结构体声明

    源代码生成器为您的状态机生成了一个部分结构体。添加您自己的部分文件：

    // File: MyStateMachineExtensions.cs
using WAYN.Games.MGM.Locomotion.Runtime.StateMachine;

namespace MyGame.Locomotion
{
    public partial struct MyStateMachine
    {
        // Override goes here
    }
}

    2. 重写管线方法

    每个执行步骤都是 ILocomotionStateMachine 上的一个 DIM。通过在部分结构体上定义方法来重写它：

    示例：被挤压时跳过积分

    public partial struct MyStateMachine
{
    public void Integrate(ref MyStateMachine stateMachine, ref MyContext context)
    {
        if (context.BaseContext.OverlapState == OverlapState.Crush)
            return; // Do nothing when crushed

        // Otherwise run the default per-state integration
        stateMachine.IntegratePerState(ref context);
    }
}

    示例：高度超过阈值时跳过地面吸附

    public partial struct MyStateMachine
{
    public void SnapToGround(ref MyStateMachine stateMachine, ref MyContext context)
    {
        // Skip ground snap when character is above a certain height
        if (context.BaseContext.CharacterPosition.Position.y > 0)
            return;
    }
}

    3. 可重写的方法

    方法 默认调用
    Execute 完整管线（所有步骤按顺序执行）
    HandleExternalImpulses 累积速度脉冲
    Integrate IntegratePerState
    SolveOverlaps 多平面反穿透
    CollideAndSlide ResolveCollisionPerState 循环
    SnapToGround 向下胶囊体投射
    InheritDisplacementFromMotionVelocity 平台速度继承
    EvaluateTransition EvaluateTransitionPerState

    完整参考请参见扩展包。

    预期结果

    您的自定义逻辑在管线的正确位置运行，包装或替换默认行为。

    常见错误

    • 只想添加前置/后置逻辑时忘记调用默认实现——该步骤会被完全跳过
    • 直接重写 IntegratePerState 或 EvaluateTransitionPerState——这些是生成的调度器，不应被重写
