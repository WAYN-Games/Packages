    创建状态机

    本指南解释如何从头创建完整的运动状态机，可以使用编辑器脚手架工具或手动创建。

    使用编辑器工具（推荐）

    通过 Unity 编辑器菜单创建状态机是最快的方式：

    1. 在 Project 窗口中右键点击您想要状态机文件夹的位置
    2. 选择 Assets > Create > WAYN Games > Locomotion > StateMachine
    3. 输入状态机的名称（例如 Vehicle、Swimming、Flying）
    4. 按 Enter

    该工具生成所有必需的文件，具有正确的命名、命名空间和关联。您可以立即开始自定义生成的代码。

    理解生成的内容

    当您创建名为 MyCharacter 的状态机时，该工具生成以下类型：

    锚点结构体

    [LocomotionStateMachine]
public partial struct MyCharacterStateMachine { }

    此结构体是状态机的核心。[LocomotionStateMachine] 属性无参数——所有类型名称按约定从结构体名称派生。对于 MyCharacterStateMachine，前缀 MyCharacter 产生：MyCharacterContext、MyCharacterProfile、MyCharacterInput、MyCharacterStateData 和 MyCharacterStates（生成的枚举）。

    状态是用 [LocomotionState] 注解的独立结构体，将它们链接回状态机：

    [LocomotionState(typeof(MyCharacterStateMachine))]
public partial struct IdleState { }

    上下文

    上下文结构体从锚点结构体源代码生成。您不需要自己编写它。生成的结构体如下所示：

    // 源代码生成——请勿手动创建
public partial struct MyCharacterContext : ILocomotionContext<MyCharacterProfile, MyCharacterInput, MyCharacterStateData>
{
    public BaseLocomotionContext BaseContext { get; set; }
    public CharacterPhysicsProfile CharacterPhysicsProfile { get; set; }
    public MyCharacterProfile Profile { get; set; }
    public MyCharacterInput Input { get; set; }
    public MyCharacterStateData StateData { get; set; }
}

    上下文是一个数据包，通过引用传递到每个状态生命周期方法。它封装了：

    • BaseLocomotionContext - 核心物理状态（位置、速度、位移、接触、重力）
    • CharacterPhysicsProfile - 烘焙的胶囊/物理配置
    • 您的自定义配置文件、输入和状态数据

    因为上下文是 partial 的，您可以在自己的 partial 声明中添加自定义属性和方法覆盖（如 ComputeGroundSnappingDistance）。

    Tip

    如果您频繁修改 BaseContext 字段，请在您的 partial 声明中将其声明为公共字段，而不是使用自动属性。这允许您使用 ref 访问：ref BaseLocomotionContext baseContext = ref ctx.BaseContext;

    配置文件

    [Serializable]
public partial struct MyCharacterProfile
{
    // 在此添加您的配置字段：
    // public float MaxSpeed;
    // public float JumpForce;
}

    ILocomotionProfile 接口由源代码生成器通过 partial 机制自动添加。配置文件数据在实体转换期间烘焙到 blob asset 中。它在运行时是不可变的，并在引用相同配置文件资源的所有实体之间共享。将所有可调的游戏参数放在这里。

    输入

    public partial struct MyCharacterInput
{
    // 在此添加您的输入字段：
    // public float3 Movement;
    // public float JumpTrigger;

    public void ApplyInputsToCharacter(in MyCharacterInput playerInputs)
    {
        // 在此复制或累积输入值
    }
}

    ICharacterLocomotionInput<MyCharacterInput> 和 IComponentData 接口由源代码生成器通过 partial 机制自动添加。输入结构体是一个 ECS 组件。它以 Update 频率从 PlayerInputAsset 接收值，并为固定更新累积它们。ApplyInputsToCharacter 方法由生成的输入读取系统在每个 Update 帧调用。

    Warning

    触发器输入（如跳跃）必须累积，而不是覆盖。如果 Update 频率高于固定更新频率，触发器按下可能会被遗漏。使用 JumpTrigger += playerInputs.JumpTrigger 累积，并在状态逻辑中通过将其重置为零来消费它。

    StateData

    public partial struct MyCharacterStateData
{
    // 在此添加每状态持久数据：
    // public int JumpCount;
    // public float StateTimer;
}

    IComponentData 接口由源代码生成器通过 partial 机制自动添加。状态数据在帧和状态转换之间持久存在。与上下文（每帧从组件重建）不同，状态数据作为 ECS 组件存储，在帧之间存续。

    添加新状态

    初始生成后，使用编辑器菜单添加状态：

    1. 在 Project 窗口中右键点击
    2. 选择为您的状态机生成的菜单项（例如 Assets > Create > WAYN Games > Locomotion > MyCharacter > State）
    3. 命名状态（例如 Swimming）

    这会生成一个用 [LocomotionState(typeof(MyCharacterStateMachine))] 注解的新状态结构体文件。源代码生成器自动将状态添加到生成的枚举中。

    或者，手动创建状态结构体并用 [LocomotionState(typeof(YourStateMachine))] 注解它。有关结构体模板，请参见编写自定义状态。

    手动创建

    如果您更喜欢手动创建所有内容（或需要了解工具做了什么），请按此清单操作：

    1. 创建一个带有 [LocomotionStateMachine] 属性的 partial struct 锚点（例如 MyCharacterStateMachine）
    2. 为配置文件数据创建 partial struct（命名为 {Prefix}Profile）
    3. 为带有 ApplyInputsToCharacter 的输入数据创建 partial struct（命名为 {Prefix}Input）
    4. 为状态数据创建 partial struct（命名为 {Prefix}StateData）
    5. 创建至少一个带有 [LocomotionState(typeof(YourStateMachine))] 的状态 partial struct
    6. 创建一个 LocomotionProfileAsset<TProfile> ScriptableObject
    7. 创建一个 PlayerInputAsset<TInput> ScriptableObject
    8. 注册泛型输入组件：[assembly: RegisterGenericComponentType(typeof(LocomotionInputAsset<TInput>))]

    源代码生成器从这些输入生成上下文结构体、状态枚举、运动系统、编写组件和所有接口实现。

    编辑器工具一键完成所有这些，这就是为什么推荐使用它。
