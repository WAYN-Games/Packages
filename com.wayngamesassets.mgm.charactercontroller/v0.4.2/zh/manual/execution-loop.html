    执行循环

    每次固定更新，运动 Job 通过精确的步骤序列处理每个角色实体。理解此序列对于编写正确的状态逻辑至关重要。

    概览

    运动 Job 作为 Burst 编译的 IJobChunk 在 FixedStepSimulationSystemGroup 中调度，在物理步骤之后。每个角色独立且并行处理。

    步骤按此确切顺序执行：

    flowchart TD
    START(["帧开始"])

    subgraph S1["1. HandleExternalImpulses"]
        HEI["从 CharacterContactSystem<br/>累积速度脉冲"]
    end

    subgraph S2["2. Integrate - <code>OnIntegrateState</code>"]
        INT["计算速度 + 位移"]
    end

    subgraph S3["3. SolveOverlaps"]
        SO["多平面去碰撞<br/>设置 OverlapState: None | Push | Pinch | Crush"]
    end

    subgraph S4["4. CollideAndSlide - <code>OnResolveStateCollisions</code>"]
        CAS_CRUSH{"OverlapState<br/>== Crush?"}
        CAS_LOOP{"HasRemaining-<br/>Displacement &&<br/>bounce < MaxBounce?"}
        CAS["投影、投射、滑动，<br/>更新接触"]
        CAS_CRUSH -- 是 --> CAS_END
        CAS_CRUSH -- 否 --> CAS_LOOP
        CAS_LOOP -- 是 --> CAS --> CAS_LOOP
        CAS_LOOP -- 否 --> CAS_END
        CAS_END([完成])
    end

    subgraph S5["5. SnapToGround"]
        SNAP_NONE{"ContactType<br/>== None?"}
        SNAP_UP{"向上移动？<br/><i>dot(vVel, gravity) < −ε</i>"}
        SNAP_VVEL{"VerticalVelocity<br/>== 0?"}
        SNAP["向下胶囊投射<br/>吸附位置 + 更新接触"]
        SNAP_NONE -- "是（跳过）" --> SNAP_END
        SNAP_NONE -- 否 --> SNAP_UP
        SNAP_UP -- "是（清除接触）" --> SNAP_END
        SNAP_UP -- 否 --> SNAP_VVEL
        SNAP_VVEL -- "否（跳过）" --> SNAP_END
        SNAP_VVEL -- "是（静止）" --> SNAP --> SNAP_END
        SNAP_END([完成])
    end

    subgraph S6["6. InheritDisplacementFromMotionVelocity"]
        IH_GROUND{"地面<br/>接触？"}
        IH_CRUSH{"挤压？"}
        IH_DYN{"站在<br/>动态实体上？"}
        IH["计算 + 应用平台位移<br/>应用扭转旋转"]
        IH_GROUND -- 否 --> IH_END
        IH_GROUND -- 是 --> IH_CRUSH
        IH_CRUSH -- 是 --> IH_END
        IH_CRUSH -- 否 --> IH_DYN
        IH_DYN -- 否 --> IH_END
        IH_DYN -- 是 --> IH --> IH_END
        IH_END([完成])
    end

    subgraph S7["7. EvaluateTransition"]
        ET["源代码生成的调度<br/>到 LocomotionTransition 结构体"]
    end

    DONE(["帧结束"])

    START --> HEI
    HEI --> INT
    INT --> SO
    SO --> CAS_CRUSH
    CAS_END --> SNAP_NONE
    SNAP_END --> IH_GROUND
    IH_END --> ET
    ET --> DONE

    管线中的每个步骤都是 ILocomotionStateMachine 上的默认接口方法（DIM），可以在您的状态机的 partial 结构体上覆盖。详情请参见自定义执行管线。

    没有 OnUpdateState 方法。积分和碰撞解析阶段共同构成状态的"更新"。转换在最后只评估一次，在所有物理解析完成之后。

    逐步分解

    1. HandleExternalImpulses

    CharacterContactSystem 在 PhysicsSimulationGroup（在接触创建和雅可比创建之间）期间运行，收集来自外部源的速度脉冲，例如：

    • 与角色碰撞的动态刚体
    • 推挤此角色的其他角色
    • 爆炸或力场

    这些脉冲在您的状态逻辑运行之前应用于角色的速度，因此您的 OnIntegrateState 看到的是更新后的速度。这就是为什么您不应该在积分开始时无条件地清除速度，除非您打算丢弃外部力。

    2. Integrate (OnIntegrateState)

    这是您的状态逻辑计算当前帧的速度和位移的地方。系统调用您的状态的 OnIntegrateState 方法。

    在此步骤中，您应该：

    • 从输入计算加速度（移动方向、跳跃力）
    • 应用重力加速度
    • 调用 baseContext.UpdateVelocitiesAndComputeDisplacements() 将速度转换为位移

    此处不应该做的事情：

    • 直接移动角色的位置（在碰撞步骤中处理）
    • 添加平台速度（ComputeInheritedDisplacement 步骤自动处理）
    • 读取本帧的接触信息（接触尚未解析；您看到的是上一帧的接触）

    运动学方程

    UpdateVelocitiesAndComputeDisplacements 应用标准运动学方程：

    displacement += velocity * dt + 0.5f * acceleration * dt * dt;
velocity += acceleration * dt;

    水平和垂直分量分别处理，允许每个分量有不同的行为（例如，重力只影响垂直方向，移动输入只影响水平方向）。

    3. SolveOverlaps

    积分后，系统检查角色的碰撞体是否与任何其他碰撞体重叠。这可能发生在：

    • 移动平台将角色推入墙壁
    • 另一个角色占据相同空间
    • 角色被传送到几何体内部

    重叠求解器尝试将角色推出重叠。如果不能（例如，角色被两个移动表面夹住），它跟踪重叠状态：

    OverlapState 含义
    None 未检测到重叠
    Push 角色已成功推出
    Pinch 角色被挤压但未完全困住
    Crush 角色完全嵌入，无法逃脱

    您可以在状态逻辑中读取 ctx.BaseContext.OverlapState 来对挤压情况做出反应（例如，杀死角色）。

    4. CollideAndSlide (OnResolveStateCollisions)

    积分计算了期望位移后，角色需要在尊重碰撞体的同时实际在世界中移动。这就是碰撞滑移循环。

    只要还有剩余位移，系统就会调用您状态的 OnResolveStateCollisions，最多 CharacterPhysicsProfile.MaxBounce 次迭代。这种多次弹跳方法允许角色沿墙壁滑动、越过障碍物并在复杂几何体中导航。

    默认行为

    如果您不覆盖 OnResolveStateCollisions，默认 DIM 调用 baseContext.Move()，执行两个操作：

    1. 位移校正 - 根据当前接触类型调整 HorizontalDisplacement 和 VerticalDisplacement：

      接触类型 行为
      None 位移按原样应用（空中自由移动）
      Ground 垂直位移被取消；水平位移沿地面坡度移动
      Wall 尝试台阶检测；如果不是台阶，位移沿墙壁表面投影
      Ceiling 垂直位移沿天花板投影；水平位移受约束

    2. 投射和移动 - 调用 PhysicsOperations.MoveUpToNextObstacle 沿校正后的位移投射胶囊，将角色移动到命中点（或如果没有命中则移动完整距离），并更新接触信息。

    要自定义碰撞解析方式，请参见自定义碰撞响应。

    台阶检测

    当检测到墙壁接触时，默认的 Move() 检查它是否可能是台阶：

    1. 角色必须与表面接触
    2. 坡度必须超过 MaxSlope
    3. 角色必须朝坡度方向移动
    4. 接触高度必须低于 MaxStepHeight

    如果所有条件都满足，PhysicsOperations.TryStep 尝试将角色移动到台阶上方。

    5. SnapToGround

    所有碰撞解析完成后，系统执行向下的胶囊投射以检测角色下方的地面表面。这确保角色在走下坡道或楼梯时保持在地面上，而不是在一帧内变为空中状态。

    吸附距离由上下文的 ComputeGroundSnappingDistance 方法计算。默认实现返回硬编码的距离 1f。在上下文的 partial 结构体上覆盖此方法以根据需要考虑台阶高度、坡度角度、速度或平台速度。

    如果在吸附距离内找到地面表面，角色的位置向下调整并更新接触信息。

    6. InheritDisplacementFromMotionVelocity

    如果角色站在动态实体（运动学或完全模拟的）上，此步骤计算角色从该实体的运动速度继承的位移和旋转。

    位移在同一帧内应用于角色的位置。继承的速度存储在：

    • baseContext.PlatformHorizontalVelocity
    • baseContext.PlatformVerticalVelocity

    这些立即可用——例如，跳跃状态的 OnEnterState（在同一帧的 EvaluateTransition 期间调用）可以读取 PlatformHorizontalVelocity 以在从平台跳跃时保持动量。来自平台移动的位置调整自动应用，这就是为什么您应该永远不要在 OnIntegrateState 中手动添加平台速度。

    7. EvaluateTransition

    最后一步评估为当前状态注册的所有 [LocomotionTransition] 结构体。此时：

    • 角色已移动到最终位置
    • 所有接触已解析且是最新的
    • 平台继承已计算

    源代码生成器按 [EvaluateBefore]/[EvaluateAfter] 属性确定的顺序调度每个转换的 ShouldTransition 方法。第一个匹配的转换获胜。生成器自动处理在当前状态上调用 OnExitState 和在新状态上调用 OnEnterState。
