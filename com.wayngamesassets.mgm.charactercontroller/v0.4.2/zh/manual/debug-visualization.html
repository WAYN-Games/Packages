    调试可视化

    MGM Character Controller 包含一个内置的调试可视化系统，可以直接在 Scene 视图中渲染接触点、移动路径和碰撞冲量。这对于诊断移动问题、调优物理参数以及理解运动系统每帧的行为非常有价值。

    启用调试可视化

    调试可视化按实体自行选择启用：

    1. 在子场景中选择角色实体的 authoring GameObject
    2. 添加 LocomotionDebugConfigAuthoring 组件
    3. 配置你想查看的可视化选项
    4. 进入 Play 模式

    调试配置被烘焙为 blob 资产，因此更改需要重新进入 Play 模式。

    调试数据类型

    路径点

    路径点显示角色在执行循环每个阶段的位置。它们可视化角色在单帧中的轨迹，包括碰撞滑动算法的每次弹跳。

    接触点

    接触点标记角色碰撞体接触环境的位置。它们按接触类型进行颜色编码：

    接触类型 含义
    Ground 表面可行走（坡度低于 MaxSlope）
    Wall 表面太陡无法行走
    Ceiling 表面在角色上方
    None 此帧无接触

    冲量点

    冲量点显示碰撞力的施加位置，包括角色接收的力和角色施加给其他物体的力。这些有助于诊断意外的速度变化或与动态对象的交互。

    调试阶段

    每个可视化点都标记了记录它的执行阶段：

    阶段 描述
    Start 任何处理之前
    SolveOverlaps 重叠解析之后
    CollideAndSlide 碰撞滑动之后
    SnapToGround 地面吸附步骤之后
    InheritDisplacement 平台速度计算之后

    这让你可以精确定位是哪个阶段产生了意外行为。

    组件和系统

    LocomotionDebugConfig

    存储每个实体调试配置的 blob 资产。通过持续时间浮点值（PathSeconds、ContactSeconds、ImpulseSeconds）控制可视化——持续时间为零或更小则禁用该类别——以及每阶段/每类型的颜色和形状设置。

    LocomotionDebugRecorder

    在作业执行期间将调试点记录到动态缓冲区中的静态工具。当调试缓冲区存在时，从运动作业内部调用。

    调试缓冲区

    三种 IBufferElementData 类型存储记录的点：

    • LocomotionDebugPathPoint - 位置 + 阶段 + 帧索引
    • LocomotionDebugContactPoint - 位置 + 法线 + 接触类型 + 物体类型 + 帧索引
    • LocomotionDebugImpulsePoint - 位置 + 方向 + 幅值 + 物体类型 + 是否施加标志 + 帧索引

    LocomotionDebugDisplaySystem

    读取调试缓冲区并使用 Unity.DebugDisplay API（通过 PhysicsDebugDisplaySystem 进行的 Burst 兼容的线条和箭头绘制）渲染可视化。

    LocomotionDebugFrameCounterSystem

    维护用于标记调试点的全局帧计数器。这允许显示系统区分不同帧的点并可选地显示历史记录。

    性能注意事项

    调试可视化会增加开销：

    • 内存：每个启用调试的实体分配三个动态缓冲区，每帧增长
    • CPU：在运动作业中记录点会增加分支检查和缓冲区写入
    • 渲染：绘制 Gizmo 在主线程上进行

    因此：

    • 只对你正在调查的实体启用调试
    • 在构建版本中禁用调试可视化
    • 当 authoring 组件不存在时，调试组件会自动排除

    自定义调试检查器

    LocomotionDebugConfigAuthoringEditor 为调试配置 authoring 组件提供自定义 Inspector。它显示已配置调试选项的视觉预览，包括持续时间设置、颜色和接触形状。

    调试技巧

    角色穿过地面

    启用路径点和地面吸附可视化。检查：

    • 地面吸附胶囊投射是否到达地面（路径点应显示向下线段）
    • 帧结束时的接触类型是否为 Ground

    角色卡在墙壁上

    启用接触点。检查：

    • 接触是被分类为 Wall 还是 Ground
    • MaxSlope 阈值是否正确
    • 台阶检测是否在尝试但失败（检查 MaxStepHeight）

    角色意外弹射

    启用冲量点。检查：

    • 是否有来自 CharacterContactSystem 的外部冲量
    • 动态物体碰撞是否注入了意外速度
    • 平台继承是否被双重应用

    状态转换未触发

    这是逻辑问题，不是物理问题。在你的转换辅助方法中添加 Debug.Log 调用（临时的，用于非 Burst 调试）或在 Entity Inspector 中检查状态机组件。

    Profiler 集成

    除了视觉调试外，该包还集成了 Unity 的 Profiler：

    ProfilerMarkerExtensions

    Burst 编译的性能分析标记工具，向 Profiler 时间线添加自定义标记。这些标记显示在运动作业的执行下方，帮助你识别哪个阶段消耗了最多时间。

    LocomotionProfilerHUD

    一个可选的 MonoBehaviour（在 Hybrid 程序集中），渲染一个简单的游戏内性能分析覆盖层，显示 FPS 和运动系统计时。
