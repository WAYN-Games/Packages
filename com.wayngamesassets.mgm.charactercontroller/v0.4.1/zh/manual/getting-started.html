    开始使用

    本指南引导您使用编辑器脚手架工具从头创建您的第一个角色。

    如果您只想快速看到包的运行效果，请改用快速入门（使用预构建的示例）。

    前提条件

    • MGM Character Controller 已安装并可编译（安装）
    • 对 Unity ECS 概念有基本了解（实体、组件、系统、烘焙）。如果您是 ECS 新手，请先查阅 Unity 的 Entities 文档。

    项目设置

    您的项目需要一个启用了物理的 sub-scene。该包依赖 Unity.Physics 进行所有碰撞检测，不使用旧版物理系统。

    1. 创建或打开一个场景
    2. 添加 Sub Scene（在 Hierarchy 中右键 > New Sub Scene）
    3. 确保 sub-scene 有 Physics Step 组件（大多数设置中自动存在）
    4. 使用 Physics Shape 组件为环境添加静态碰撞体（地板、墙壁等）

    创建您的第一个角色

    最快的入门方式是使用内置脚手架工具：

    步骤 1：生成状态机

    1. 在 Project 窗口中，右键点击您想要创建状态机的文件夹
    2. 选择 Assets > Create > WAYN Games > Locomotion > StateMachine
    3. 为文件夹命名（例如 MyCharacter）
    4. 按 Enter

    该工具生成一个包含锚点文件的最小文件夹结构。其他所有内容在编译时由源代码生成：

    MyCharacter/
├── Runtime/
│   ├── mycharacter.runtime.asmdef               # 自动生成的程序集定义
│   └── MyCharacterStateMachine.cs               # 带 [LocomotionStateMachine] 属性的锚点结构体
└── Authoring/
    └── mycharacter.authoring.asmdef             # 自动生成的程序集定义

    编译后，源代码生成器在内存中生成所有剩余类型（上下文、状态枚举、配置文件、输入、状态数据、编写组件、运动系统、输入读取系统）。然后通过 Assets > Create > WAYN Games > Locomotion 子菜单添加状态、转换和数据结构体。

    步骤 2：创建配置文件资源

    1. 在 Project 窗口中右键点击
    2. 选择为您的角色配置文件资源生成的 Create 菜单项（菜单路径已为您生成）
    3. 在 Inspector 中配置移动速度、跳跃力、重力乘数和其他参数

    步骤 3：创建输入资源

    1. 在 Project 窗口中右键点击
    2. 选择为您的玩家输入资源生成的 Create 菜单项
    3. 分配移动和跳跃的 Input Action References

    步骤 4：设置角色预制件

    1. 在 sub-scene 中创建一个新的 GameObject
    2. 添加生成的 Character Locomotor Authoring 组件
    3. 分配您的 Profile Asset
    4. 配置 Physics Profile（胶囊高度、半径、质量、最大坡度等）

    步骤 5：设置玩家预制件

    Player 是与 Character 不同的概念。玩家拥有输入并控制一个或多个角色。

    1. 在 sub-scene 中创建一个 GameObject（它被烘焙为 Player 实体）
    2. 添加生成的 Player Character Authoring 组件
    3. 分配 Controlled Character（Locomotor 的编写组件）、Input Asset 和 Player GO Prefab（包含摄像机和输入组件的常规 GameObject 预制件，保存在 sub-scene 外部）

    步骤 6：进入 Play 模式

    按 Play。您的角色应该响应输入并与物理环境交互。如果角色穿过地板，请验证：

    • 您的地板有物理碰撞体
    • 角色的碰撞层（BelongsTo 和 CollidesWith）配置正确
    • Sub-scene 已加载

    导入示例

    该包包含一个 SinglePlayerSample，演示了完整的设置，包括五个状态（Idle、Moving、Jumping、Falling、Sliding）、摄像机装置、移动平台和调试可视化。

    要学习示例：

    1. 导航到 Assets/MGM Locomotion/Samples/SinglePlayerSample/
    2. 打开 Scenes/ 中的场景
    3. 查看 Scripts/SinglePlayerStateMachine/ 中的脚本以获取完整的参考实现
    Tip

    示例是理解所有部分如何连接的最佳资源。请在阅读本文档的同时阅读它。
